## 3. 히어링에서 물어야 할 질문과 기대 답변

단순히 정보를 모으는 게 아니라, **"이 답이 오면 다음에 이렇게 움직인다"** 는 로직을 사전에 준비한다.

### 3-1. 예산감에 관한 질문

**묻는다：** 「현재 보수비 외에, 새 시스템에 어느 정도의 초기 투자를 생각하고 계신가요?」

| 답변 패턴 | 다음 액션 |
|----------|---------|
| "500만엔 이내라면" | 단계적 개발 플랜 제안, 페이즈 1만으로 수렴 |
| "가능하면 무료로" | 사내 프로젝트화해서 회사 부담 형태 제안 |
| "먼저 써보고 나서가 아니면" | 시험 도입 1지점 소규모 플랜을 먼저 제안 |
| "얼마든지 좋으니 빨리 바꾸고 싶어" | 풀스펙·단기 납품 플랜 제안 |

---

### 3-2. 의사결정에 관한 질문

**묻는다：** 「이 시스템 도입을 최종 결정하는 분은 누구신가요? 또 어떤 조건이 갖춰지면 결단하실 수 있나요?」

| 답변 패턴 | 다음 액션 |
|----------|---------|
| "제가(담당자가) 결정할 수 있어요" | 오늘 자리에서 구체적 다음 단계를 합의 |
| "사장님 확인이 필요해요" | 사장용 1장 요약 자료를 별도 작성 |
| "다른 시스템이랑 비교하고 싶어요" | 경쟁사 비교표를 만들어 우위성 명시 |
| "먼저 시험 도입 보고 나서" | 시험 도입 조건·평가 기준을 함께 정함 |

---

### 3-3. 리스크에 관한 질문

**묻는다：** 「시스템 도입에 있어서 가장 걱정되는 점은 무엇인가요?」

| 답변 패턴 | 대응해야 할 자료 |
|----------|--------------|
| "스태프가 쓸 수 있을지" | 조작 매뉴얼·연수 플랜 준비 |
| "데이터 안 사라지는지" | 백업·복구 구조를 도식화 |
| "중간에 개발 안 멈추는지" | 프로젝트 계속의 담보 방법 명시 |
| "VitaminC보다 불편해지는 거 아닌지" | 기능 대응표 + 데모로 불안 해소 |
| "비용이 불어나지 않는지" | 고정비 모델 제안(변동비 없음) |

---

### 3-4. 기존 데이터에 관한 질문

**묻는다：** 「현재 고객 정보나 매출 데이터는 어떤 형태로 보관되어 있나요? 건수는 어느 정도인가요?」

| 답변 패턴 | 대응 |
|----------|------|
| "VitaminC DB에 있어요(건수 불명)" | VitaminC 데이터 익스포트 형식 조사 |
| "엑셀로도 관리하고 있어요" | CSV 가져오기 기능을 우선 구현 목록에 추가 |
| "수만 건이나 있어요" | 이행 기간·이행 테스트를 계획에 포함 |
| "백업이 없어요" | 우선 기존 데이터 백업부터 지원 |

---

### 3-5. 시험 도입에 관한 질문

**묻는다：** 「만약 1~2개 지점에서 먼저 해보신다면, 어디가 후보가 될까요?」

| 답변 패턴 | 대응 |
|----------|------|
| "규모 작은 지점에서" | 소규모 지점용 간이 버전부터 시작 |
| "스태프가 젊은 지점에서" | 연수 불필요한 직관적 UI 우선 |
| "본부 직영점에서" | 본부 기능(리포트)을 처음부터 구현 |
| "선정부터 상의하고 싶어요" | 선정 기준을 함께 책정하는 프로세스 제안 |
