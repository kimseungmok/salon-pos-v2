## 3. ヒアリングで聞くべき質問と期待する回答

単に情報を集めるだけでなく、**「この答えが来たら次にこう動く」** というロジックを事前に準備する。

### 3-1. 予算感に関する質問

**聞く：** 「現在の保守費以外に、新システムに対してどの程度の初期投資をお考えですか？」

| 回答パターン | 次のアクション |
|------------|-------------|
| 「500万円以内なら」 | 段階的開発プランを提案、フェーズ1だけで収める |
| 「できれば無料で」 | 社内プロジェクト化して会社負担の形を提案 |
| 「まず試してからでないと」 | 試験導入1店舗の小規模プランを先に提案 |
| 「いくらでもいい、早く変えたい」 | フルスペック・短期納品プランを提案 |

---

### 3-2. 意思決定に関する質問

**聞く：** 「このシステムの導入を最終的に決める方は誰でしょうか？また、どんな条件が揃えば決断できますか？」

| 回答パターン | 次のアクション |
|------------|-------------|
| 「私（担当者）が決められる」 | 今日の場で具体的な次のステップを合意する |
| 「社長に確認が必要」 | 社長向けの1枚サマリー資料を別途作成 |
| 「他のシステムと比較したい」 | 競合比較表を作成して優位性を明示 |
| 「まず試験導入を見てから」 | 試験導入の条件・評価基準を一緒に決める |

---

### 3-3. リスクに関する質問

**聞く：** 「システム導入にあたって、一番心配していることは何ですか？」

| 回答パターン | 対応すべき資料 |
|------------|-------------|
| 「スタッフが使えるか」 | 操作マニュアル・研修プランを用意 |
| 「データが消えないか」 | バックアップ・復旧の仕組みを図解 |
| 「途中で開発が止まらないか」 | プロジェクト継続の担保方法を明示 |
| 「VitaminCより不便にならないか」 | 機能対応表＋デモで払拭 |
| 「コストが膨らまないか」 | 固定費モデル提案（変動費なし） |

---

### 3-4. 現行データに関する質問

**聞く：** 「現在の顧客情報や売上データはどんな形で保存されていますか？件数はどのくらいでしょうか？」

| 回答パターン | 対応 |
|------------|------|
| 「VitaminCのDBにある（件数不明）」 | VitaminCのデータエクスポート形式を調査 |
| 「Excelでも管理している」 | CSV取込機能を優先実装リストに追加 |
| 「何万件もある」 | 移行期間・移行テストを計画に含める |
| 「バックアップがない」 | まず現行データのバックアップから支援 |

---

### 3-5. 試験導入に関する質問

**聞く：** 「もし1〜2店舗でまず試してみるとしたら、どこが候補になりますか？」

| 回答パターン | 対応 |
|------------|------|
| 「規模が小さい店舗で」 | 小規模店舗向けの簡易版から開始 |
| 「スタッフが若い店舗で」 | 研修不要な直感的UIを優先 |
| 「本部直営店で」 | 本部機能（レポート）を最初から実装 |
| 「テスト店舗の選定から相談したい」 | 選定基準を一緒に策定するプロセスを提案 |
