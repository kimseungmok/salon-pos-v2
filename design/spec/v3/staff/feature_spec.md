# スタッフ (직원) — 기능 정의서

> 대상 화면: 11(スタッフ一覧), 12(スタッフ詳細), 13(シフト管理), 33(アカウント招待)
> 근거: `toss_benchmarking.md` §1-4, §1-16, 토스플레이스 가이드 `guide/sector/prepaid-store/payment/connect-sajangnim-book`

---

## F-STAFF-00. 범위 확정 (중요)

**토스플레이스에는 직원 권한/PIN/역할 관리 화면 자체가 존재하지 않는다.** 4개 업종(선불형/후불형/서비스업/도소매업) 전체를 검색해도 "직원", "권한관리" 관련 문서화된 화면이 없음을 확인(`toss_benchmarking.md` §1-4, §1-16). 토스가 제공하는 직원 관련 기능은 **딱 하나**:

> "매출장부 연동할 직원 초대하기" — 이름+휴대폰번호 입력 → 알림톡 발송 → 동의하면 사장님 매출장부 앱에 매출이 연동됨. 상태는 `待機中`/`連結済み` 2가지뿐.

따라서:
- **33(アカウント招待)** = 토스 기능 그대로 구현(F-STAFF-01)
- **11(スタッフ一覧), 13(シフト管理)** = 토스에 대응 화면 없음 → **살롱 고유 자산으로 명시 유지**(시술 담당자 배정·근무표 관리는 살롱 운영에 필수이므로 자체 설계)
- **12(スタッフ詳細)** = PIN/권한 변경은 POS 범위 밖(통근관리사이트 전용)이라는 안내만 표시, 나머지(기본정보/시술이력 등)는 살롱 고유

---

## F-STAFF-01. 직원 초대 (33)

### 규칙
- 입력: 이름 + 휴대폰번호만(로그인ID/PIN/권한레벨 **입력란 자체를 두지 않음** — 과거 v2 구현에서 제거 완료, v3에서도 동일 원칙 유지)
- 초대 발송 시 알림톡(또는 SMS) 자동 발송, 상태 `待機中`
- 피초대자가 동의하면 `連結済み`로 전환, 이후 매출 데이터 연동(연동 범위는 본 정의서 범위 외 — 인프라 영역)
- `再送信`(대기중 상태에서만), `削除`(언제든) 액션 제공

---

## F-STAFF-02. 스태프 목록/상세 (11/12, 살롱 고유)

- 11: 시술 담당자로서의 기본 정보(이름/연락처/소속점/근무상태) 리스트 — 권한·PIN 컬럼 없음
- 12: 기본정보 + 시술 실적(담당 매출, 가동률 등 — Fresha 참고 보너스) + "PIN・権限レベルの変更は勤怠管理サイト（管理者専用）で行います。" 안내 고정 문구

---

## F-STAFF-03. 시프트 관리 (13, 살롱 고유)

- 주간/월간 근무표, 담당자별 근무시간 블록
- 06(予約カレンダー)의 "담당자 가용여부" 판정과 데이터 공유(F-BOOK-02의 担当者칩 "空き/休み" 표시는 13의 시프트 데이터를 참조해야 함 — **교차 의존**)
