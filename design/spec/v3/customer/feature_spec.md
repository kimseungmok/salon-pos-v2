## F-CUST-01. 고객 그룹 자동 분류

### 정의
고객은 사장님이 수동으로 등급을 지정하지 않는다. **방문 이력에 따라 시스템이 자동으로 4단계 그룹 중 하나로 분류**한다. 토스 실제 기준을 그대로 따른다.

### 그룹 정의 및 판정 규칙

| 그룹 | 아이콘 | 판정 조건 | 우선순위 |
|---|---|---|---|
| 初回来店 (첫방문) | 🐣 | 누적 방문 횟수 = 1 (오늘 첫 방문 포함) | 4 (최후순위 — 다른 조건 불충족시 기본값) |
| 予備常連 (예비단골) | 🟡 | 누적 방문 횟수 ≥ 2 이고, 常連 조건 미달 | 3 |
| 常連 (단골) | 🔴 | **최근 90일 이내 방문 7회 이상** | 1 (최우선 — 항상 먼저 검사) |
| 休眠ぎみ (뜸한고객) | ⚪ | (예비단골 또는 단골 조건 충족) **이고** 최근 방문일로부터 45일 이상 경과 | 2 (常連보다 늦게, 나머지보다 먼저 검사) |

### 판정 알고리즘 (우선순위 순서로 검사)

```
groupOf(customer):
  daysSinceLastVisit = today - customer.lastVisitDate
  visits90d = countVisits(customer, withinDays=90)
  totalVisits = customer.visits.length

  if totalVisits == 1:
      return 초회来店          # 무조건 첫방문 (다른 조건보다 먼저 확정)
  if daysSinceLastVisit >= 45:
      return 休眠ぎみ           # 단골/예비단골이었다가 멀어진 경우
  if visits90d >= 7:
      return 常連
  return 予備常連
```

> **주의**: 토스 원본은 업종별로 기준이 다르다(카페는 "한 달 평균 2회 이상" 등). 본 정의서는 **헤어살롱 기준**으로 "최근 90일 7회 이상"을 채택한다 — 평균 12~13일에 1회 방문하는 정기 고객을 常連로 잡기 위함.

### 예외 케이스

| 케이스 | 처리 |
|---|---|
| 노쇼(ノーショー)만 있고 실제 방문 0회 | 그룹 분류 대상에서 제외하거나 別途 "ノーショー履歴"로만 표시. 첫방문 그룹에 포함하지 않음. |
| 당일 신규 등록 + 당일 예약(미방문) | 初回来店로 분류하되 "来店予定"임을 명시 |
| 환불/노쇼로 시술이 취소된 방문 | 방문 횟수 카운트에서 제외 |
| 그룹 재계산 시점 | 방문 완료(체크아웃) 처리 시 즉시 재계산. 일배치 재계산은 불필요(휴면 전환은 조회 시점에 동적 계산하면 충분) |

## F-CUST-02. 고객 메모

### 정의
고객 상세화면 상단에 **항상 노출되는 메모 입력란**. "顧客についてのメモを書いてください。オーナーのみ見られます"(고객에 대한 메모를 작성해주세요. 사장님만 볼 수 있어요) 안내 placeholder 고정.

### 규칙
- 권한: 사장님(오너) 계정만 작성/조회 가능 — 일반 스태프 계정에는 메모란 자체를 숨김 처리 검토 필요(현재 목업은 권한분기 미구현, v3 신규 화면에서 보강 권장)
- 글자수 제한 없음(토스도 제한 없음으로 확인)
- 저장 시점: 입력 필드 blur 또는 일정 디바운스(예: 1.5초) 후 자동저장

## F-CUST-03. 포인트 / 선불권 표시

고객 상세화면 Zone C에 2개의 카드로 표시한다.

| 카드 | 표시 내용 | 액션 |
|---|---|---|
| ポイント | 보유 포인트 숫자(P) | −/＋ 버튼으로 수동 가감 가능 |
| プリペイド券 | 보유 잔액 합계 또는 "なし" | ＋ 버튼 → **선불권 신규 구매/충전 플로우 진입** (상세는 `prepaid_pass/feature_spec.md` 참조) |

> 기존 목업(`10_customer_detail.html`)의 プリペイド券 "＋" 버튼은 toast만 띄우는 자리표시자였음 — v3에서 실제 플로우로 연결 필요(F-PP 영역 참조).

## F-CUST-04. 고객 검색 / 리스트 필터

### 정의
09 화면 상단 검색창은 **번호 뒷자리 또는 이름**으로 검색. 그룹 탭(すべて/初回来店/予備常連/常連/休眠ぎみ)으로 필터링.

### 규칙
- 검색 대상: 전화번호 뒷 4자리 일치 또는 이름 부분일치
- 정렬 기본값: 최근 방문일 내림차순(recent)
- 그룹 탭 카운트는 실시간으로 F-CUST-01 알고리즘 재계산 결과를 반영(캐싱하지 않음 — 데이터 갱신 즉시 카운트 변경)

## F-CUST-05. 고객 상세 — 통계/그래프

- 来店回数(총 방문 횟수), 最終来店(마지막 방문일), 誕生日(생일)을 요약 통계로 표시
- 그래프 영역은 토스 근거 없는 자체 보너스 기능(방문 추이 등) — 살롱 고유 가치로 유지, 데이터 정의서에서 별도 표기
