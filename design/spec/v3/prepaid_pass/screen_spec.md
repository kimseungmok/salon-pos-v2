## 화면 27 — プリペイド券管理（선불권 메뉴 관리, 신규）

토스의 "선불권 관리하기" 진입 화면 + 메뉴 생성 폼을 통합한 화면. 25(商品リスト)와 자매 화면으로, 사이드바 "商品・在庫管理" 하위에 배치.

### Zone 구조

| Zone | 내용 |
|---|---|
| Zone B | 타이틀 `プリペイド券管理` + 우측 `＋プリペイド券作成` 버튼 |
| Zone C | 생성된 선불권 메뉴 카드 리스트(금액권/횟수권 구분 표시) |
| 모달 | 생성/수정 모달(F-PP-01 입력 항목 그대로) |

### Zone C 카드 레이아웃

```
[색상스와치] 10万円チャージ券          金額チャージ券
             ¥100,000 → ¥110,000チャージ（+10%ボーナス）
             有効期限：チャージ日から1年            [編集] [無効化]
```

```
[色] カット10回券                      回数券（適用：カット）
     ¥80,000 / 10回                      残り使用可能：たくさんのお客様が保有中
     有効期限：チャージ日から6ヶ月          [編集] [無効化]
```

### 마이그레이션 모달 (F-PP-05, 결정사항)

Zone B 우측에 `紙の回数券・利用券を移行` 버튼 추가:

```
紙の回数券・利用券を移行
顧客を選択 *           [検索]
種類                   [回数券] [金額券]
残り回数／金額          [____]
元の購入日 *           [____年__月__日]   ← purchasedAt（유효기간 계산 기준）
                            [キャンセル]   [プリペイド券として登録]
```

### 생성 모달 (F-PP-01 매핑)

```
① 유형          [金額チャージ券] [回数券]   ← 토글, 생성 후 잠금
② 메ニュー名     [______________]
③ (回数券のみ) 適用商品   [商品を選択 1つだけ +]
④ (回数券のみ) 回数       [____] 回
⑤ 決済価格       [______] 円   □ 毎回直接入力する
⑥ チャージ方式   [決済した分だけチャージ] [追加でボーナスを付与]
   └ ボーナス選択 시: 追加金額/回数 입력 필드 노출
⑦ 有効期限       [なし][3ヶ月][6ヶ月][1年][2年][3年][日付指定][直接入力]
                 ※期限30日前+当日に無料アルリムトーク発送
        [キャンセル]                          [作成する]
```

- ①에서 回数券 선택 시 ③④가 나타나고 ⑤의 라벨이 "決済価格"로 고정(금액권은 "チャージ金額"로 표기 차이 없음, 동일 필드 재사용)
- 생성 완료 시 토스트: `「{メニュー名}」を作成しました`

## 화면 28 — プリペイド券チャージ／使用モーダル（충전·사용, 신규）

02/03(POS) 및 10(顧客詳細)에서 공통으로 호출하는 모달. 호출 위치에 따라 진입 시 고객 지정 여부만 다르다.

### 모드 A — チャージ（충전, F-PP-02）

```
顧客を選択 *                [検索: 電話番号下4桁 または 名前]
─────────────────────────────
チャージするメニューを選択
 ○ 10万円チャージ券  ¥100,000 → ¥110,000
 ○ カット10回券      ¥80,000 / 10回
─────────────────────────────
決済方法            [現金][カード][QR決済][銀行振込]
─────────────────────────────
            [キャンセル]          [¥110,000を決済]
```

- "一般商品と同時決済不可"(F-PP-02) — 카트에 다른 상품이 이미 있으면 이 모달 진입 시 경고: `プリペイド券は一般商品と別々に決済する必要があります。先に他の商品の決済を完了してください。`
- 결제 완료 시: `¥{price}決済完了` + 고객 정보 패널에 `プリペイド券 ¥{amount} 1件` 형태로 이력 누적

### 모드 B — 使用（사용, F-PP-03）— 02/03 POS 결제수단 그리드에서 호출

```
決済方法に「プリペイド券」を追加（既存グリッドの1枠として）
   🎫
 プリペイド券

선택 시 하위 화면:
─────────────────────────────
{顧客名}様の保有プリペイド券
 ◉ 10万円チャージ券   残高 ¥94,500（89日後失効）
 ○ カット10回券       残り 6回（180日後失効）
─────────────────────────────
今回の決済金額 ¥5,500 のうち
 プリペイド券から ¥5,500 を使用 → 残り¥0 を他の方法で決済

            [キャンセル]          [プリペイド券で決済]
```

- 잔액 부족 시(F-PP-03 혼합결제 규칙): "今回の決済金額のうちプリペイド券から¥Xを使用 → 残り¥Yを他の方法で決済" 형태로 자동 분할 안내, 확인 후 나머지는 기존 결제수단 그리드로 이동
- 결제 완료 토스트: `¥{used}使用完了 — {menuName}（残高¥{remaining}、{days}日後失効）`

## 10(顧客詳細) 화면 연동 변경점

기존 `feature_spec.md` F-CUST-03 / 기존 목업 대비 변경:

```diff
- <div class="pc-val muted">なし</div>
- <button class="pc-btn solid" onclick="toast('プリペイド券発行（モック）')">＋</button>
+ <div class="pc-val">¥94,500（残高）/ カット6回</div>   <!-- 보유 시 -->
+ <button class="pc-btn solid" onclick="openPrepaidModal(customerId, 'charge')">＋</button>
```

`openPrepaidModal()`은 화면28(모드A)을 고객 지정된 상태로 오픈.
