## F-PP-00. 선불권이란 (정의)

방문할 때마다 결제할 필요 없이, **미리 서비스 이용금액을 결제해두고 방문할 때마다 차감하는 방식**. 매장 특성에 따라 두 가지 유형 중 하나로 만들어 사용한다.

| 유형 | 차감 방식 | 예시 |
|---|---|---|
| 金額チャージ券 (금액권) | 사용한 **금액만큼** 차감 | 10万円券, 20万円券 |
| 回数券 (횟수권) | 사용한 **횟수만큼** 차감, **특정 메뉴 1개에만 연결** | カット10回券, パーマ20回券 |

---

## F-PP-01. 선불권 메뉴 생성

매장 사장님이 판매할 선불권 "메뉴"를 미리 만들어 둔다(상품 등록과 유사한 개념).

### 공통 입력 항목

| 항목 | 필수 | 설명 |
|---|---|---|
| 유형 | ✅ | `금액차감` 또는 `횟수차감` — 생성 후 변경 불가(타입 전환 금지) |
| 메뉴이름 | ✅ | 예: "10万円チャージ券", "カット10回券" |
| 결제가격 | ✅ | 고객이 실제로 지불할 금액. "매번 직접 입력할게요" 체크 시 결제 시점에 가격을 매번 입력(가격 미고정형) |
| 적용상품 | 횟수권만 ✅ | **1개 상품만 선택 가능**. 해당 메뉴에서만 횟수 차감됨 |
| 횟수 | 횟수권만 ✅ | 1회 구매 시 제공되는 횟수(예: 10) |
| 충전방식 | ✅ | 아래 F-PP-01a 참조 |
| 사용기한 | 선택 | 아래 F-PP-01b 참조 |

### F-PP-01a. 충전 방식(보너스 충전)

| 옵션 | 동작 |
|---|---|
| 결제한 만큼만 충전해요 | 지불액(또는 횟수)만큼 정확히 충전 |
| 추가 금액(횟수)을 더 드려요 | 보너스 추가 — 예: ¥100,000 결제 시 ¥110,000 충전, 또는 10회 구매 시 12회 충전. 보너스 비율/금액은 메뉴 생성 시 고정값으로 입력 |

### F-PP-01b. 사용기한

옵션: `없음` / `3개월(90일)` / `6개월(180일)` / `1년` / `2년` / `3년` / `날짜지정` / `직접입력`

- 사용기한은 **충전(구매)한 실제 날짜로부터** 계산 — 메뉴 생성일이 아님
- 만료 30일 전 + 만료 당일, 고객에게 **무료 알림톡(LINE/SMS 등 채널) 자동 발송**

### 제약

- 한 매장에서 동일 적용상품에 횟수권을 여러 개 만들 수 있음(가격대별로 10회권/20회권 등)
- 이미 발행된(고객이 보유한) 선불권의 유형·적용상품은 변경 불가 — 새 메뉴를 만들어야 함

## F-PP-02. 선불권 충전(판매)

### 정의
고객이 선불권을 구매(충전)하는 결제 행위. 일반 상품 결제와 흐름은 비슷하나 다음 차이가 있다.

### 규칙

| 규칙 | 내용 |
|---|---|
| 일반 상품과 동시 결제 불가 | 선불권과 일반 상품(시술/제품)은 **반드시 별도 결제**로 처리. 카트에 선불권과 일반상품이 같이 담기면 결제 버튼 비활성 또는 분리 결제 안내 |
| 분할결제 불가 | 선불권 결제 자체는 분할결제(F-PAY 분할결제 기능) 대상에서 제외 |
| 결제 취소 시 재사용 불가 | 충전 결제를 취소(환불)하면 해당 충전분은 그대로 소멸 — "다시 담기(재시도)" 불가. 환불 시 잔액에서도 동일 금액/횟수를 함께 차감 처리 필요 |
| 고객 식별 필수 | 충전 전 반드시 고객을 지정해야 함(전화번호 검색 또는 신규 등록) |
| 고객 동의 | 해당 고객이 "고객관리 서비스 이용 동의"를 한 적이 없다면 최초 1회 동의 절차 필요(알림톡 수신 등을 위함) |

### 충전 경로 (진입점 2곳)

1. **포스 홈 → [선불권] 탭**: 메뉴 선택 → 고객 지정 → 결제수단 선택 → 결제
2. **고객 상세화면(10) → プリペイド券 카드 [+]**: 고객이 이미 지정된 상태로 진입 → 메뉴 선택 → 결제

## F-PP-03. 선불권 사용(결제 차감)

### 정의
시술/상품 결제 시 결제수단으로 "선불권"을 선택하면 보유 잔액에서 차감.

### 규칙

| 규칙 | 내용 |
|---|---|
| 잔액 부족 시 | 부족분은 다른 결제수단(현금/카드 등)과 **혼합 결제 가능**(선불권 우선 차감 + 나머지 별도 결제) — 토스 원본은 이 부분이 명확히 문서화되어 있지 않으므로, 본 정의서는 "잔액 한도까지 차감 후 나머지는 별도 결제수단 선택"으로 운영 규칙을 확정한다 |
| 횟수권 적용 범위 | 해당 메뉴(F-PP-01에서 지정한 적용상품)에만 사용 가능. 다른 메뉴 결제에는 사용 불가 |
| 사용 완료 표시 | "¥N 사용 완료" + "선불권명(잔액 ¥N, N일 남음)" 토스트 표시 |
| 동시 보유 | 고객 1명이 여러 선불권(금액권+횟수권, 또는 횟수권 여러 개)을 동시에 보유 가능. 사용 시 결제 화면에서 선택 |

---

> **참고(교차검증)**: 予約金(F-BOOK-02a)은 1회 예약의 보증금이고, 본 영역의 선불권은 여러 차례 충전·사용 가능한 잔액이다 — 두 개념은 독립적이며 서로 차감/전환되지 않는다.

## F-PP-04. 만료 처리

- 만료된 선불권은 잔액이 남아있어도 더 이상 사용/충전 불가 — 목록에서 `失効` 상태로 표시
- 만료 30일 전 알림(F-PP-01b) 발송 후, 매장이 직접 연장 처리할 수 있는 수단은 토스 가이드에 명시되어 있지 않음(추가 조사 필요 항목으로 `toss_benchmarking.md`에 남김)

## F-PP-05. 과거 종이/실물 티켓 이행 (마이그레이션, 결정사항)

`payment_pos/feature_spec.md` F-PAY-02 결정에 따라 결제수단에서 回数券/利用券을 제거하고 선불권으로 통합한다. 기존에 종이로 발행했던 회수권·이용권을 보유한 고객은 27 화면에서 일괄 등록해 `PrepaidPassBalance`로 전환한다.

- 27(プリペイド券管理)에 `紙の回数券・利用券を移行` 액션 추가(상세 화면 정의는 `screen_spec.md` 참조)
- 이행 시 원본(종이) 잔여 횟수/금액을 그대로 `remainingCount`/`remainingAmount`에 입력, `purchasedAt`은 이행 처리일이 아니라 **원래 구매일**(고객 확인 후 입력)로 기록해 유효기간 계산이 왜곡되지 않게 함
