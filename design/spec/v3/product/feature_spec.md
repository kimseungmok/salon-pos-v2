# 商品・カテゴリ (상품/카테고리) — 기능 정의서

> 대상 화면: 25(商品リスト), 26(商品登録フォーム → 25로 흡수)
> 근거: `toss_benchmarking.md` §1-5, §1-14, 토스플레이스 가이드 `guide/start-here/product-category-setup`

---

## F-PROD-01. 상품 등록은 모달 방식 (전체페이지 아님)

토스 원본은 상품 등록을 **모달**로 처리한다(별도 페이지 이동 없음). v2 단계에서 이미 26번 화면을 25에 흡수해 리다이렉트 전용 파일로 축소한 결정을 v3에서도 그대로 승계한다.

### 모달 입력 항목

| 항목 | 설명 |
|---|---|
| 색상 스와치 | 11색 팔레트(백색 포함) 중 1개 선택 — **사진 업로드가 아니라 색상 스와치가 기본**. 이 색이 02(注文)의 메뉴 타일 배경색이 됨(F-PAY-01과 연동, 카테고리 단위로 고정 권장) |
| キオスク露出 | 토글 ON/OFF — 무인 키오스크 화면에 노출할지 여부 |
| 商品名 | 텍스트 |
| カテゴリ | **칩 단일선택**(여러 개 동시 소속 불가) + "새 카테고리 등록" 링크 |
| 価格 | 숫자 + "매번 직접 입력할게요"(시가상품, F-PROD-03) 체크 |
| オプション | **칩 다중선택**(길이/코스추가/색농도/사이즈/본수추가 등) |

## F-PROD-02. 카테고리 관리

- 1열 리스트(서브탭) + 키오스크노출 토글 + 상단 `순서 편집` / `영어표기` / `+カテゴリ追加`(블루 솔리드)
- 카테고리에 색상을 고정하면 해당 카테고리의 모든 상품 타일이 동일 색으로 보임(F-PAY-01 권고사항과 직결 — **카테고리:색상 = 1:1 고정 매핑**을 데이터 모델에서 강제)

---

## F-PROD-03. 시가 상품 (가격 매번 직접 입력)

- "매번 직접 입력할게요" 체크 시 POS에서 해당 상품 선택할 때마다 가격 입력 모달이 뜸
- 살롱에서는 옵션/추가시술료가 매번 달라지는 케이스(예: 디자인펌 난이도별 추가요금)에 사용

---

## F-PROD-04. 재고 — 범위 확정 (중요)

**토스 전체(서비스업+도소매업)에 수량 기반 재고 추적 대시보드가 없음**(`toss_benchmarking.md` §1-14). 도소매업의 "바코드 스캐너 사용하기"도 상품 등록 자동화 + 주문 수량 카운터일 뿐이다. 따라서:

- 14(在庫現況)/15(在庫変動履歴) = 토스에 대응 기능 없는 **살롱 고유 자산**으로 명시 유지(이미 §1-14에서 배지 처리 완료)
- F-PROD-01의 상품 등록 모달에는 "재고수량" 입력 필드를 굳이 추가하지 않음 — 재고는 14/15에서 별도 관리(상품과 1:N 또는 1:1 연동만 유지)
