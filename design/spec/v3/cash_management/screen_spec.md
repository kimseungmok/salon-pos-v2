# レジ金管理 (시재관리) — 화면 정의서

> 기능 규칙은 `feature_spec.md` 참조(F-CASH-xx로 인용).

---

## 화면 22 — 開店準備（개점준비）

### Zone 구조

| Zone | 내용 |
|---|---|
| Zone B | 타이틀 `開店準備` + 날짜 네비게이션(‹ 날짜 › 今日) + 우측 `開店確定` 버튼 |
| 좌 컬럼 | 本日のスタッフ在店状況（F-CASH-03） |
| 우 컬럼 | 現在のレジ金額（開始金カウント）— F-CASH-02 |

### 우 컬럼 상세

```
現在のレジ金額（開始金カウント）
┌────────────────────────┐
│ 総額            ¥21,500 │  ← 큰 숫자, 실시간 갱신
└────────────────────────┘
金種        枚数・個数        小計
¥10,000   [-] [ 2] [+] 枚    ¥20,000
¥5,000    [-] [ 0] [+] 枚    ¥0
¥1,000    [-] [ 0] [+] 枚    ¥0
¥500      [-] [ 3] [+] 個    ¥1,500
¥100      [-] [ 0] [+] 個    ¥0
...
────────────────────────────
合計                          ¥21,500
前日締め予想額                 ¥50,000
差額                          −¥28,500（赤字）
　差額があります。再確認してください。
```

- `開店確定` 버튼: 총액 > 0일 때만 활성화

## 화면 23 — レジ締め（폐점）

### Zone 구조

동일 패턴 + 좌 컬럼이 매출 요약(本日の売上)으로 대체, 하단에 체크리스트(F-CASH-04) 추가.

```
本日の売上
実売上 ¥XXX,XXX ・ 注文件数 N件 ・ 返品 −¥X,XXX
決済手段別売上 [현금/카드/QR... 바 그래프]

────────────────────────
現在のレジ金額（締めカウント）
（22와 동일한 권종별 입력 UI, "総額" 큰 숫자 표시 포함）
合計          ¥XX,XXX
予想締め額     = 開始金 + 現金売上 − 現金返金
差額           ...

────────────────────────
閉店チェックリスト（F-CASH-04）
☐ 本日の売上精算書を出力・確認
☐ 施術スペース・待合スペースを整理
☐ 消耗品・在庫の締め確認
☐ 出入口施錠・セキュリティON

                              [閉店確定]  ← 전체 체크 완료 시만 활성
```
