## F-BOOK-01. 예약 캘린더 표시 (06)

### 규칙
- 좌측: 월간 미니캘린더(일요일 빨강, 토요일 파랑, 선택일 진한 파랑 원)
- 우측: 담당자별 컬럼(컬러닷+이름), 시간축 세로
- 예약 블록: **담당자 컬러를 흰색 62% 비율로 혼합한 파스텔 배경 전체채움**(화이트바디+컬러스트립 아님). 블록 안에 고객이름(볼드)+시술명
- 노쇼: 블록에 작은 "ノーショー" 칩 추가
- 현재시각 라인: 파란 가로선 + 시각 라벨, 매분 갱신
- 블록 클릭 → 팝업: 아바타+이름+그룹(F-CUST-01 결과)+"총N회 방문, 노쇼N회"+포인트/시술정보+메타(담당자/일시/메모)+우하단 floating `+`(새 예약 추가 진입점)

## F-BOOK-02. 예약 등록 (07)

### 입력 흐름
1. 고객 검색(기존) 또는 신규 등록
2. **예약금(F-BOOK-02a)** — 토글 ON/OFF
3. 날짜·시간(시작~종료, 메뉴 합계 소요시간으로 자동 산출, 수동 조정 가능)
4. 메뉴(칩, +추가)
5. 담당자 지정(칩, 해당 시간대 가능 여부 표시: 空き/予約あり/休み)
6. 추가 설정: メモ, LINE通知送信, 定期予約(반복), 事前承認가 필요

### F-BOOK-02a. 예약금(デポジット)
- ON 시: 결제수단(銀行振込/カード) + 금액 + `受け取りました` 체크 + 환불안내문구(텍스트, 기본값 "返金は24時間以内に可能です。")
- OFF 시: 예약금 없이 등록만(워크인 예약과 동일 취급)
- 예약금 ON 상태로 저장 시 → 카카오톡(또는 LINE) 스타일 알림톡 미리보기 패널에 입금 안내 자동 표시(토스 원본 기준, **본 정의서 범위에서는 텍스트 안내문 생성까지만 정의**, 실제 메시징 연동은 별도 인프라 영역)
- **선불권(F-PP)과는 별개 개념**: 예약금은 1회 예약에 대한 보증금, 선불권은 여러 차례 사용 가능한 충전형 잔액 — 혼동 주의

### 규칙
- 담당자 미지정("미지정") 허용 — 당일 배정
- 정기예약(반복) 설정 시 동일 패턴으로 N건 예약을 일괄 생성(주1회/2주마다/매월)

## F-BOOK-04. 예약 취소/노쇼 시 예약금 처리 (결정사항, 미해결 이슈 해소)

`payment_pos/feature_spec.md` F-PAY-05(결제취소 원자적 처리)와 동일한 패턴을 예약금에도 적용한다.

### 처리 규칙

| 상황 | 예약금 처리 |
|---|---|
| 고객이 사전에 취소(시술 시작 24시간 전 — F-BOOK-02a `refundNote` 기본 정책과 일치) | **전액 환불** |
| 고객이 임박 취소(24시간 이내) 또는 **노쇼(ノーショー)** | **환불 불가 — 매장 보상금으로 귀속**(전액 매출로 전환 처리) |
| 매장 사정으로 취소(스태프 휴진 등) | **전액 환불**(귀책사유 매장) |

- 매장은 예약 등록 시(F-BOOK-02a `refundNote`)에 위 기준과 다른 정책을 자유 텍스트로 명시할 수 있다 — 위 표는 **기본값**이며 강제 규칙은 아니다
- 환불 처리는 F-PAY-05와 동일하게 **원자적**으로 처리: 예약금 결제(Payment)와 Booking.status 갱신이 동시에 성공해야 함(부분 실패 방지)
- 환불은 결제 시 사용한 동일 수단(은행이체/카드)으로 처리(별도 결제수단 환불 불가 — F-PAY 원본 결제수단 환불 원칙과 동일)

## F-BOOK-03. 웨이팅 관리 (08) — 토스 근거 없음, 살롱 고유 기능

### 정의
워크인(예약 없이 방문) 고객을 대기열로 관리. **토스플레이스 전체에 대응 기능이 없음**(전체 가이드 검색으로 확인, `toss_benchmarking.md` §1-13) — "대기 예약"이라는 용어는 F-BOOK-02a의 예약금 미입금 상태를 가리키는 전혀 다른 개념이므로 혼동하지 않는다.

### 규칙
- 상태: 待機中 → 呼び出し済み → (완료는 履歴로 이동) / 取消 가능
- 대기 순번은 등록 순서 자동 부여, 수동 순서변경(▲▼) 허용
- 대기시간 경과에 따라 색상 경고(10분미만/10~20분/20분이상)
- 평균 대기시간 통계는 당일 데이터로 실시간 산출
