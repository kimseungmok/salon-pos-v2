# 決済・POS — 화면 정의서

## 화면 02 — 注文（메뉴 선택 / 카트）

### Zone 구조

| Zone | 내용 |
|---|---|
| Zone A | 상단바(공통) |
| Zone B(좌) | 카테고리 탭 + 메뉴 타일 그리드 |
| Zone C(우) | 카트 패널 |

### 메뉴 타일

```
┌──────────────┐
│ カット        │  ← 배경 전체 = 카테고리 색상(예: #8E44AD)
│              │
│ ¥5,000 ・40分│
└──────────────┘
```

- 타일 배경색: `TILE_COLORS` 순환 팔레트(카테고리별 고정 색상 권장 — 매번 인덱스로 순환하면 같은 메뉴가 다른 색으로 보일 위험, v3에서는 **카테고리 ID에 색상을 고정 매핑**하도록 개선)
- 품절: 우하단 `品切れ` 배지(반투명 검정), 탭 비활성
- 재고 배지(선택적, F-PAY-01 보수적 처리): 좌상단 또는 우상단 작은 숫자 — 표시 여부는 카테고리 설정에서 토글 가능하게(예: 소모품 판매용 메뉴만 노출)

### 카트 패널

```
[수량-][1][수량+]  カット                      ¥5,000
[수量-][1][수량+]  ワンカラー                  ¥8,000
────────────────────────────────
[割引]              [分割決済]
        ¥13,000 決済
```

## 화면 03 — 決済方法選択・分割決済

### 비분할결제 모드

```
総決済金額
¥13,000を決済します

クーポン・ポイント・プリペイド券を使用 ›

決済方法
┌──────┬──────┬──────┐
│現金   │カード │QR決済 │
├──────┼──────┼──────┤
│銀行振込│後払い │プリペイド券│ ← F-PAY-02 신규 추가 항목
└──────┴──────┴──────┘
```

- `プリペイド券` 선택 시 화면28(모드B)로 전환(고객 미지정 상태면 먼저 고객 검색 요구)
- `現金` 선택 시 인라인 영역 노출(F-PAY-02a): 받은금액 입력 + 빠른버튼 + 거스름돈 표시줄

### 분할결제 모드 (F-PAY-04)

```
[金額で決済] [割り勘] [メニュー別決済]   ← 세그먼트 탭

(金額で決済 선택 시)
総決済金額 ¥13,000のうち
今回の決済金額
[______] 円
1 2 3
4 5 6
7 8 9
00 0 ←
                          [キャンセル]  [確認]

→ 확인 후 결제수단 선택 그리드로 이동 → 결제 → "¥N決済完了 / 残り¥M（総¥13,000）" 토스트
→ 잔여금액 있으면 같은 화면에서 추가 결제 반복
```

- `メニュー別決済` 선택 시: 카트 아이템에 체크박스 노출, 선택된 항목 합계만큼 결제
