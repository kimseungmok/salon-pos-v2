## F-PAY-01. 메뉴 타일 표시 (02)

### 규칙
- 카테고리 탭(흰 배경, 활성 탭은 밑줄)으로 메뉴를 분류
- 상품 타일은 **카테고리 등록 시 지정한 색상 스와치가 배경 전체를 채움**(화이트카드+컬러스트립 아님). 타일 위에 이름(좌상단)·가격(좌하단)·소요시간을 흰 텍스트로 표시
- 품절 상품: 우하단에 검정 반투명 "品切れ" 배지, 탭 불가
- 타일 우상단 작은 숫자 배지: 재고/잔여 수량으로 추정(토스 가이드에 문서화된 화면은 없으나 스크린샷에서 일관되게 확인됨 — §1-16 참조). **확정 근거 부족 — 구현 시 "있으면 표시, 없으면 숨김" 정도로 보수적으로 처리하고 별도 재고 대시보드와는 연결하지 않음**

### 카트 패널(우측)
- 아이템 리스트: 수량 ±, 메뉴명, 가격
- 하단 버튼: `割引` / `分割決済` (F-PAY-04 참조) + 전체폭 결제 버튼 `¥{total}決済`
- 매장/포장/둘다 세그먼트는 **외식업 전용 — 살롱에는 미적용**(여러 위치/대기열 개념 없음)

## F-PAY-02. 결제수단 선택 (03)

### 정의
결제 버튼 클릭 시 진입하는 화면. 총 결제금액 표시 + 쿠폰/포인트/선불권 사용 링크 + 결제방법 그리드.

### 결제수단 종류 (그리드, 아이콘+라벨)

| 수단 | 설명 |
|---|---|
| 現金 | 받은금액 입력 → 거스름돈 자동계산(F-PAY-02a) |
| カード | 단말기 연동(모킹 시 즉시 성공 처리) |
| PayPay / LINE Pay | QR결제 — 일본 현지 주요 결제앱 반영(토스 원본은 韓国 QR결제였으나 현지화) |
| 後払い | 미수금 처리(매출장부에 외상으로 기록) |
| 掛け売り | 거래처 매입 계상(법인고객 대상, 살롱 고유 확장) |
| **プリペイド券** | **F-PP-03(선불권 사용) 연동**. 클릭 시 화면28(모드B)로 진입 |

> **결정사항(미해결 이슈 해소, 2026-06-23)**: 回数券/利用券은 결제수단 그리드에서 **제거하고 プリペイド券으로 통합**한다. 종이/실물로 이미 발행된 과거 티켓이 있다면 `27_prepaid_pass.html`에서 잔여 횟수·금액을 선불권으로 일괄 이행(migration) 처리하고, 결제수단은 항상 プリペイド券 하나로만 관리한다 — 이중관리 제거.
> 기존 v2 목업에는 プリペイド券이 결제수단 그리드에 없었으므로, v3에서는 반드시 그리드에 추가하고 F-PP-03 플로우로 연결한다(이행 후에도 동일).

### F-PAY-02a. 현금 결제 — 거스름돈 계산
- 받은금액 입력칸(인라인, 항상 공간 확보) + 빠른금액 버튼(정확히/올림/¥5,000/¥10,000)
- 거스름돈 = 받은금액 − 결제금액, 음수면 결제 버튼 비활성

### 쿠폰/포인트 적용
- `クーポン適用` 칩 → 쿠폰 선택 모달(F-MKT-01, 19번 화면 참조)
- `ポイント使用` → 보유 포인트 한도 내 사용액 입력

## F-PAY-03. 결제 완료 처리 (04)

- 결제 성공 시 영수증/전자영수증 발행 여부 선택(個人/事業者 또는 自進発行/未発行)
- 적용된 쿠폰/포인트/선불권 차감을 모두 확정(트랜잭션 커밋)

## F-PAY-04. 분할결제 (03 내)

### 정의
한 주문을 여러 번/여러 사람이 나눠 결제. 3가지 방식 중 선택.

| 방식 | 설명 |
|---|---|
| 金額で決済 | 임의의 금액을 입력해 그만큼만 결제, 남은 금액은 추가 결제로 처리 |
| 割り勘 | 인원수를 지정하면 총액을 균등 분배(나머지는 첫 결제자가 흡수 등 규칙 필요 — 구현 시 결정) |
| メニュー別決済 | 카트의 특정 메뉴만 선택해 그만큼 결제 |

### 규칙
- 분할결제 진행 중 결제수단은 매 회차마다 다르게 선택 가능
- **선불권은 분할결제 대상에서 제외**(F-PP-02 제약과 일치)
- 잔여 금액이 남아있는 동안 "총 결제금액 N원 중 M원 남음" 안내 + 추가 결제 진행
- 전액 결제 완료 시 F-PAY-03으로 이동

---

## F-PAY-05. 거래 내역 조회 (05)

- 결제완료/취소/분할결제 이력을 시간순으로 조회
- 결제 취소(환불) 시 F-PP-02(선불권 환불), F-CUST(포인트 환원) 등 연관 트랜잭션도 함께 롤백되어야 함 — **취소 처리는 단일 트랜잭션으로 원자적 처리** 필요(부분 실패 방지)
