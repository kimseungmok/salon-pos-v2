# 05 · 取引履歴 — 화면 정의서

> **파일**: `design/mockups/v2/ja/05_transaction_history.html`
> **목적**: 결제 완료된 거래 내역 조회 + 취소/환불 처리를 한 화면에서 수행. 기존에 분리 계획했던 "05 거래이력(취소/환불)"과 "18 거래이력(매출)"을 하나로 통합.
> **진입**: 사이드바 → 売上 → 取引履歴 / 04_pos_complete 완료 화면에서 "履歴で確認"
> **이동**: 거래 클릭 → Zone D 상세 패널에 표시 → 全額返金/部分返金 처리
> **컨셉**: 마스터-디테일 (좌측 리스트 + 우측 영수증형 상세 패널)

---

## 1. 구역 명명 (Zone Map)

```
┌─────────────────────────────────────────────────────────────────────────┐
│  Zone A  グローバルトップバー                              (전체 폭 1행)  │
├──────────────────────────────────────────────────────┬──────────────────┤
│  Zone B  ツールバー (기간 탭 · 검색)                    │                 │
├──────────────────────────────────────────────────────┤  Zone D         │
│                                                      │  詳細パネル      │
│  Zone C  取引リスト                                    │  (영수증형)      │
│          (거래 카드 목록, 세로 스크롤)                  │                 │
│                                                      │  미선택 시:      │
│                                                      │  今日の売上状況  │
└──────────────────────────────────────────────────────┴──────────────────┘
```

| 구역 | CSS 클래스 | 역할 | 크기 |
|------|-----------|------|------|
| Zone A | `.zone-a` | 앱 전역 헤더 | H 44px · 전체 폭 |
| Zone B | `.zone-b` | 기간 탭 전환 · 검색 | H 46px · 좌측 폭 |
| Zone C | `.zone-c` | 거래 리스트 (스크롤) | flex:1 |
| Zone D | `.zone-d` | 상세/통계 패널 | W 320px |

## 2. Zone B — ツールバー

| 요소 | 설명 |
|------|------|
| タイトル | "取引履歴" |
| 期間タブ | `今日` `今週` `今月` `全期間` — 세그먼트 탭 |
| 検索ボックス | 고객명 / 取引番号 검색 |

---

## 3. Zone C — 取引リスト

### 3-1. 카드 구성

```
┌──────────────────────────────────────────────────────────────┐
│ 10:05  田中 美咲                          ¥14,300   [カード]  │
│        カット＋カラー　他1件                       完了 ●     │
└──────────────────────────────────────────────────────────────┘
```

| 요소 | 설명 |
|------|------|
| 시각 | HH:MM (오늘 외 날짜는 `M/D HH:MM`) |
| 고객명 | 클릭 시 Zone D에 상세 표시, 선택된 카드 강조(인디고 보더) |
| 품목 요약 | 첫 항목명 + `他N件` (2건 이상 시) |
| 금액 | 환불 발생 시 원금액에 취소선 + 환불 후 금액 별도 표시 |
| 결제수단 배지 | 現金💴 / カード💳 / PayPay📱 / LINE Pay💬 / 後払い📋 |
| 상태 배지 | `完了`(녹) / `部分返金`(황) / `返金済み`(적) |

### 3-2. 상태 색상 (00_design_system C-04 준용)

| 상태 | 색상 | 배지 텍스트 |
|------|------|-------------|
| `completed` | `#1E8449` | 完了 |
| `partial_refund` | `#D4AC0D` | 部分返金 |
| `refunded` | `#E74C3C` | 返金済み |

---

## 4. Zone D — 詳細パネル

### 4-1. 거래 선택 시 (영수증형)

| 요소 | 설명 |
|------|------|
| 헤더 | 고객명·전화번호, 取引番号, 日時, 担当スタッフ |
| 품목 리스트 | 항목명 + 금액 (부분환불 시 환불된 항목에 취소선) |
| 小計 / 合計 | 04_pos_complete 영수증 스타일 재사용 |
| 支払方法 | 배지 |
| 상태 배지 | 完了/部分返金/返金済み |
| **返金 액션** (완료 상태일 때만) | `全額返金` (위험 버튼) / `部分返金` (보조 버튼) |
| 返金フォーム | 액션 클릭 시 인라인 펼침: 부분환불은 품목별 체크박스, 사유 입력(필수), 확인/취소 버튼 |
| 返金情報 (환불 처리됨) | 환불액·사유·처리시각 표시, 액션 버튼 숨김 |

### 4-2. 미선택 시 — 今日の売上状況

| 항목 | 설명 |
|------|------|
| 総売上 | 오늘 완료 거래 합계(환불액 차감) |
| 取引件数 | 오늘 거래 건수 |
| 返金件数 / 返金額 | 오늘 발생한 환불 건수·금액 |
| 決済手段別件数 | 미니 범례 (현금/카드/QR 등 비율) |

## 5. JS 상태 관리

| 변수/함수 | 역할 |
|-----------|------|
| `TXNS[]` | 거래 데이터 (id, datetime, daysAgo, customer, phone, items[], total, method, staff, status, refund{amount,reason,at,items[]}) |
| `curPeriod` | `'today'\|'week'\|'month'\|'all'` |
| `selectedId` | 현재 Zone D에 표시 중인 거래 id (null이면 통계 표시) |
| `switchPeriod(p)` | 기간 탭 전환 |
| `selectTxn(id)` | 상세 패널 표시 |
| `openRefundForm(type)` | `'full'\|'partial'` 환불 폼 토글 |
| `toggleRefundItem(idx)` | 부분환불 품목 체크 토글 |
| `confirmRefund()` | 상태 갱신(완료→부분환불/환불됨), 토스트, 통계 재계산 |
| `render()` | 검색/기간 조건 반영해 Zone C, Zone D 갱신 |
