# 00 · 디자인 시스템 정의서

> **파일**: `design/mockups/ja/00_design_system.html`  
> **목적**: 전체 앱에서 사용되는 색상·타이포·스페이싱·컴포넌트의 단일 레퍼런스. 이 문서를 기반으로 모든 화면이 일관성을 유지함.  
> **진입 경로**: 개발자·디자이너 전용 참고 문서. 앱 내 실제 화면 아님.  
> **작성 배경**: 35개 모크업을 만들기 전 공통 기반을 먼저 정의해 스타일 불일치를 방지.

## TAB-06 · 실제 토스플레이스 화면 검증 (2026-06-23)

`tossplace.gitbook.io/guide` 실제 가이드 스크린샷(주문화면·예약화면·상품등록 모달·포인트 설정 등)을 직접 열어 대조한 결과 기록. 지금까지 "토스 100%"라고 표기한 화면들은 **기능/필드 구성은 실제 토스와 거의 일치**했지만, 시각 디자인 일부가 달랐던 것을 확인하고 보정 시작.

### V-01 · 확인된 것 (기존 디자인이 실제와 부합)
| 항목 | 실제 토스 | salon-pos-v2 | 결론 |
|------|----------|--------------|------|
| 상단바 색 | 다크 슬레이트 네이비 | `#1C2833` | 거의 일치 — 변경 불필요 |

### V-03 · TILE_COLORS 팔레트 (실제 색상 스와치 순서 기준 추정치)
```
#5B8DEF(블루) #6FCF73(그린) #F2994A(오렌지) #F0C75E(옐로우) #A78BFA(퍼플)
#9AA5B1(그레이) #4FD1C5(틸) #3B5BDB(네이비) #A0714F(브라운) #F2A0C0(핑크)
```
`02_pos_order.html`의 `TILE_COLORS`/`tileColor(idx)`로 구현, 상품 인덱스 기준 순환 적용. 향후 예약 블록(06)·재고 카드(14/25) 등에도 동일 팔레트 재사용 권장.

### V-04 · 남은 작업 (우선순위순)
1. `06_booking_calendar` 예약 블록 → 단색 파스텔 배경으로 전환 (TILE_COLORS 재사용)
2. 전체 화면 공통 햄버거 메뉴를 2열 아이콘 그리드로 전환 (`02/03/04/05/06/07/08/09/10/11/13/14/17/19/20/21/22/23/25` 등 21개 파일 영향)
3. `26_product_form` 사진 업로드 → 색상 스와치 선택 + 모달 구조로 전환, 카테고리/옵션을 칩 다중선택으로 전환
4. 25/14 등 리스트형 화면의 카드도 톤 점검(필요시 파스텔 적용)

규모가 크므로 한 세션에 전부 끝내지 않고 화면별로 순차 진행 중. 진행 상황은 이 섹션에 계속 업데이트.
