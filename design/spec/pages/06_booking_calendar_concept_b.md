# 06-B · 예약 캘린더 — Concept B · 단일 타임라인 레이아웃

> **파일**: `design/mockups/v2/ja/06_booking_calendar_b.html`  
> **컨셉 키워드**: "지금 이 시간에 무슨 일이 일어나고 있나"  
> **설계 원칙**: 세로축=시간, 예약 카드가 시간을 점령하는 시각. 스태프 구분은 색과 배지로.

---

## 1. 언제 이 레이아웃을 쓰는가

| 상황 | 이유 |
|------|------|
| 혼자 운영하거나 스태프가 2명 이하 | 열이 적어서 컬럼 뷰 효과가 반감됨 |
| 예약 겹침(오버랩) 파악이 중요한 살롱 | 동일 시간대 블록이 나란히 표시되어 충돌이 시각화됨 |
| "지금 몇 시, 뭐가 진행 중인가" 빠른 파악 | 스크롤 없이 NOW 라인 주변만 보면 됨 |
| 예약 카드에 정보를 최대한 넣고 싶을 때 | 열 분할 없이 가로 폭 전체를 카드가 사용 |

## 3. 주요 구성 요소

### 3-1. 스태프 범례 패널 (좌측 100px)
- 각 스태프 이름 + 색 도트 + 당일 예약 건수
- 휴무 스태프는 opacity 0.35
- 하단 고정 영역: **▶ NOW** 레이블 + 현재 시각 (14px 굵은 폰트)
- 탭 형식의 필터 역할은 없음 (필터는 상단 툴바 칩으로)

### 3-2. 타임라인 그리드 (우측 메인)
- 좌측 46px = 시간 레이블 (9:00–19:00)
- 정각선: `#EBEBEB` / 30분선: `#F4F4F2`
- 1분 = `PX_PER_MIN` (1.4px) → 1시간 = 84px (컨셉 A 40px보다 더 세밀)
- 배경: 짝수 시간대는 `#FAFAF8`으로 미묘하게 교차 (줄무늬 없음)

### 3-3. 예약 카드 — 가장 핵심적인 차이점

**겹침 처리 (오버랩 감지)**
```
시간이 겹치는 예약들은 JS로 감지 →
같은 시간대의 카드들을 가로로 등분해서 배치

예: 10:30–12:00에 佐藤(Yuki), 加藤(Kenji)가 겹치면
    왼쪽 50% = 佐藤 카드
    오른쪽 50% = 加藤 카드
```

**카드 구성 (세로 순서)**
```
┌─┬──────────────────────────────────────┐
│ │  田中 美咲    [Yuki]               │  ← 고객명 + 스태프 배지
│ │  カット＋カラー                      │  ← 메뉴명
│ │  09:00〜10:00             ¥14,300  │  ← 시간 범위 + 금액
│ │  [完了]                             │  ← 상태 배지
└─┴──────────────────────────────────────┘
↑
스태프 색 왼쪽 바 (4px)
```

**카드 배경**: 거의 흰색 (`rgba(255,255,255,0.94)`) — 밝고 읽기 쉬움  
**높이별 표시 규칙**:

| 카드 높이 | 표시 항목 |
|-----------|-----------|
| 28px 이상 | 고객명 + 스태프 배지 |
| 36px 이상 | + 메뉴명 |
| 52px 이상 | + 시간 범위 + 금액 |
| 70px 이상 | + 상태 배지 |

### 3-4. NOW 라인
- 빨간 가로선 (`#EF4444`, 2px) + 좌측 원형 마커
- 좌측에 `10:24` 텍스트 레이블 (::before)
- 스태프 범례 패널 하단의 시계와 연동되는 시각 효과

## 4. 뷰 전환

### 일(日) 뷰 — 메인 (위 구조)
- 타임라인 방식. 겹침 자동 처리.

### 주(週) 뷰
- 컨셉 A와 동일한 7열 그리드.
- 예약 블록은 스태프 컬러 바(3px) + 고객명 + 스태프명으로 표시.
- 당일(水) 열에는 실제 예약 블록 표시, 나머지는 빈 그리드 (목업 데이터 한계).

### 월(月) 뷰
- 컨셉 A와 동일한 6×7 그리드.
- 셀 내 예약 바 3개까지 표시, 초과분은 `+N件`.

### 일람(一覧) 뷰
- 컨셉 A보다 카드 왼쪽에 스태프 컬러 바(3px) 강조.
- 카드: 스태프 바 | 시간 | 고객명 + 메뉴 | 금액 + 스태프 배지 | 상태 배지.

## 6. 컨셉 B의 강점 vs 약점

**강점**
- 같은 시간대에 "무슨 일이 일어나고 있나"가 즉시 보임
- 카드 폭이 넓어서 고객명·메뉴·금액·상태를 모두 표기 가능
- 겹침 감지로 더블 부킹·시간 충돌이 시각화됨
- 혼자 운영하거나 스태프가 적은 살롱에 적합
- NOW 라인 주변을 보면 현재 진행 상황을 즉시 파악

**약점**
- 스태프별 빈 슬롯 파악이 어려움 ("Mika가 14시에 비어 있나?"는 가로 스캔 필요)
- 겹침이 많으면 카드가 좁아져 정보가 다시 잘림
- 스태프 3명 각각이 몇 시에 뭘 하는지 비교는 컬럼 뷰가 더 명확

## 7. 컨셉 A vs B 비교표

| 항목 | Concept A · 스태프 열 | Concept B · 타임라인 |
|------|----------------------|---------------------|
| 주축 | 가로=사람, 세로=시간 | 세로=시간 (사람 구분은 색) |
| "누가 비어 있나" | ✅ 열 스캔 한 번 | ❌ 어렵 |
| "지금 뭐가 진행 중" | 보통 (가로 스캔 필요) | ✅ 즉시 |
| 카드 정보량 | 열 폭에 제한됨 | ✅ 넓어서 풍부 |
| 겹침 시각화 | ❌ 열이 달라서 안 보임 | ✅ 나란히 배치 |
| 스태프 5명+ | 열이 너무 좁아짐 | 무관 |
| 혼자 운영 살롱 | 열 하나라 낭비 | ✅ 적합 |
| 팀 살롱 (3–4명) | ✅ 최적 | 보통 |
