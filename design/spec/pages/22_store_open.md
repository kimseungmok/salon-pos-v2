# 22 · 開店準備 — 화면 정의서

> **파일**: `design/mockups/v2/ja/22_store_open.html`
> **목적**: 영업 시작 전 체크리스트 확인 + 시재(현금) 시작금 카운트. 토스플레이스 가이드의 "시재 관리" 공백을 메우는 화면.
> **진입**: 사이드바 → 店舗設定 → 開店準備 (영업일 최초 진입 시 자동 유도 가능)
> **이동**: 開店確定 클릭 → `02_pos_order` (영업 시작)
> **컨셉**: 좌측 체크리스트+출근현황 / 우측 시재 카운트

---

## 1. 구역 명명 (Zone Map)

```
┌─────────────────────────────────────────────────────────────────────────┐
│  Zone A  グローバルトップバー                                            │
├─────────────────────────────────────────────────────────────────────────┤
│  Zone B  ツールバー (날짜 · 開店確定 버튼)                                │
├──────────────────────────────────┬──────────────────────────────────────┤
│  Zone C  チェックリスト            │  Zone D  レジ開始金カウント           │
│  Zone C-1 開店チェック             │  권종별 수량 입력 → 합계              │
│  Zone C-2 出勤状況                 │  예상 시작금과 차액 비교               │
└──────────────────────────────────┴──────────────────────────────────────┘
```

| 구역 | 역할 |
|------|------|
| Zone A | 앱 전역 헤더 |
| Zone B | 영업일 날짜 표시, 開店確定 버튼 (조건 미충족 시 비활성) |
| Zone C-1 | 개점 전 점검 체크리스트 (4항목) |
| Zone C-2 | 금일 출근 스태프 현황 (출근/휴무 토글) |
| Zone D | 권종별(지폐·동전) 수량 입력 → 시작금 합계, 전일 마감 예상액과 차액 표시 |

---

## 2. Zone D — レジ開始金カウント (핵심)

| 권종 | 입력 |
|------|------|
| ¥10,000 / ¥5,000 / ¥1,000 (지폐) | 수량 입력 → 단가×수량 자동 계산 |
| ¥500 / ¥100 / ¥50 / ¥10 / ¥5 / ¥1 (동전) | 〃 |
| **合計** | 전체 합산 |
| **前日締め予想額** | 고정값 표시 (mock: ¥50,000) |
| **差額** | 합계−예상액. 0이면 녹색 "일치", 아니면 적색 강조 + 경고 문구 |

開店確定 버튼은 체크리스트 4항목 전부 체크 + 차액이 확인된(0 또는 사용자가 "차액 인지하고 진행" 선택) 상태에서만 활성화.

---

## 3. JS 상태 관리

| 변수/함수 | 역할 |
|-----------|------|
| `CHECKLIST[]` | 점검 항목 (id, label, checked) |
| `STAFF_ATTEND[]` | 스태프별 출근 상태 |
| `DENOMS[]` | 권종(value, label, qty) |
| `EXPECTED_START` | 전일 마감 예상 시작금 (mock 고정값) |
| `toggleCheck(id)` / `toggleAttend(name)` | 체크/출근 토글 |
| `updateQty(value, qty)` | 권종 수량 입력 → 합계·차액 재계산 |
| `canOpen()` | 開店確定 활성화 조건 판정 |
| `confirmOpen()` | 개점 확정 처리, 토스트, (실제 구현 시 02_pos_order 이동) |
