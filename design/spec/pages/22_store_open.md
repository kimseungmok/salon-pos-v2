# 22 · 開店準備 — 화면 정의서

> **파일**: `design/mockups/v2/ja/22_store_open.html`
> **목적**: 영업 시작 전 시재(현금) 시작금 카운트 + 본일 스태프 체크인 현황 확인.
> **진입**: 사이드바 → 店舗設定 → 開店準備 (영업일 최초 진입 시 자동 유도 가능)
> **이동**: 開店確定 클릭 → `02_pos_order` (영업 시작)
> **컨셉**: 좌측 지점 체크인 현황(로그) / 우측 시재 카운트
> **변경 배경**: 디자이너가 하루 안에 여러 지점을 오가며 헬프하는 운영 형태를 반영해, 단순 "개점 체크리스트 + 출근/휴무 토글"을 폐기하고 `06_booking_calendar`와 동일한 **체크인 모델**(PIN 로그인=체크인, 타지점 재로그인=자동 체크아웃+이동)을 그대로 노출하는 화면으로 전환.

---

## 1. 구역 명명 (Zone Map)

```
┌─────────────────────────────────────────────────────────────────────────┐
│  Zone A  グローバルトップバー                                            │
├─────────────────────────────────────────────────────────────────────────┤
│  Zone B  ツールバー (날짜 · 開店確定 버튼)                                │
├──────────────────────────────────┬──────────────────────────────────────┤
│  Zone C  本日のスタッフ在店状況     │  Zone D  レジ開始金カウント           │
│  (체크인 로그)                     │  권종별 수량 입력 → 합계              │
│                                  │  예상 시작금과 차액 비교               │
└──────────────────────────────────┴──────────────────────────────────────┘
```

| 구역 | 역할 |
|------|------|
| Zone A | 앱 전역 헤더 |
| Zone B | 영업일 날짜 표시, 開店確定 버튼 (시재 입력 전까지 비활성) |
| Zone C | 본일 이 지점에 체크인한 스태프 로그 (읽기 전용, 자동 집계) |
| Zone D | 권종별(지폐·동전) 수량 입력 → 시작금 합계, 전일 마감 예상액과 차액 표시 |

---

## 2. Zone C — 本日のスタッフ在店状況

각 행에 표시:
- 스태프 색 도트
- 이름 + (타지점 소속이면) `○○店よりヘルプ` 배지
- 체크인~체크아웃 시간대 텍스트:
  - 재실 중: `09:02 〜 在店中`
  - 퇴실(타지점 이동) 완료: `09:10 〜 13:00（池袋店へ移動）`
- 상태 배지: `在店中`(녹) / `退店済み`(회색)

체크리스트·출근/휴무 토글은 **제거**됨. 근태(휴식시간 규정, 정사원/파트 시급 등) 관리는 별도 근태관리 사이트(외부 시스템) 영역으로 분리하고, POS는 "누가 몇시~몇시에 이 지점에 있었는가"만 기록·표시한다.

---

## 3. Zone D — レジ開始金カウント

| 권종 | 입력 |
|------|------|
| ¥10,000 / ¥5,000 / ¥1,000 (지폐) | 수량 입력 → 단가×수량 자동 계산 |
| ¥500 / ¥100 / ¥50 / ¥10 / ¥5 / ¥1 (동전) | 〃 |
| **合計** | 전체 합산 |
| **前日締め予想額** | 고정값 표시 (mock: ¥50,000) |
| **差額** | 합계−예상액. 0이면 녹색 "일치", 아니면 적색 강조 + 경고 문구 |

開店確定 버튼은 시재 합계가 입력되면(0보다 크면) 활성화. 차액이 있어도 진행 가능하되 경고 문구는 유지.

## 4. JS 상태 관리

| 변수/함수 | 역할 |
|-----------|------|
| `CHECKINS[]` | 본일 체크인 로그 (name, color, homeBranch, checkIn, checkOut, movedTo) |
| `THIS_BRANCH` | 현 지점명 (체크인 기록과 비교해 ヘルプ 여부 판정) |
| `DENOMS[]` | 권종(value, label, qty) |
| `EXPECTED_START` | 전일 마감 예상 시작금 (mock 고정값) |
| `renderCheckins()` | Zone C 로그 렌더링 |
| `updateQty(value, qty)` | 권종 수량 입력 → 합계·차액 재계산 |
| `updateOpenBtn()` | 開店確定 활성화 조건 판정 (시재 입력 여부만) |
| `confirmOpen()` | 개점 확정 처리, 토스트, (실제 구현 시 02_pos_order 이동) |
