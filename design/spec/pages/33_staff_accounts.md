# 33 · アカウント・権限管理 — 화면 정의서

> **파일**: `design/mockups/v2/ja/33_staff_accounts.html`
> **목적**: 전 계정(ID/PIN/소속/직급/상태)을 테이블로 일괄 관리 + 권한 매트릭스를 직접 토글 편집.
> **진입**: 사이드바 → システム設定 → アカウント・権限管理
> **컨셉**: **계정관리 전용화면(A2)**. `11/12`의 통합형(A1)과 비교 검토용.

---

## 1. 구역 구성

| 구역 | 역할 |
|------|------|
| Zone B | 타이틀 + ＋アカウント発行 |
| 계정 테이블 | 스태프·ログインID·PIN(마스킹+표시)·소속·권한레벨(인라인 select)·상태·無効化 액션 |
| 권한 매트릭스 | 4단계×6기능, **토글 스위치로 직접 ON/OFF**. オーナー열은 항상 잠김(변경불가) |

## 2. A1 vs A2 비교 포인트 (요약)

`12_staff_detail.md` §3 참조. 핵심 차이: A2는 권한을 **편집 가능**(토글), A1(12)은 **읽기 전용**(해당 직급 강조만). 실제 운영에서는 권한 변경은 보안상 민감하므로 A2처럼 오너 전용 별도 화면으로 분리하는 편이 안전할 가능성이 있음 — 향후 결정 필요.
