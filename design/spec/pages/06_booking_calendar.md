# 06 · 予約カレンダー — 화면 정의서

> **파일**: `design/mockups/v2/ja/06_booking_calendar.html`  
> **목적**: 예약 관리 중심 화면. 일/주/월/일람 4개 뷰로 예약 현황 시각화 및 예약 추가·편집·상태 처리·대기 관리 수행.  
> **진입**: 사이드바 → 予約カレンダー / 글로벌 탑바 탭  
> **이동**: 예약 추가·클릭 → `07_booking_form` / 결제 → `02_pos_order`  
> **컨셉**: 레이아웃 A (스태프 열 구조) + 디자인 B (카드형 블록)

## 1. 구역 명명 (Zone Map)

화면을 아래 6개 구역으로 정의한다. 이후 지시·스펙·코드 리뷰 시 이 이름으로 참조한다.

```
┌─────────────────────────────────────────────────────────────────────────┐
│  Zone A  グローバルトップバー                              (전체 폭 1행)  │
├──────────────────────────────────────────────────────┬──────────────────┤
│  Zone B  カレンダーナビバー                            │                 │
├──────────────────────────────────────────────────────┤  Zone F         │
│  Zone C  スタッフセレクター                            │  サイドパネル    │
├────────────────────────────────────────────────────  │                 │
│  Zone D  スタッフコラムヘッダー                        │  F-1 ミニカレ   │
├──────────────────────────────────────────────────────┤  F-2 日次統計   │
│                                                      │  F-3 ウェイティング│
│  Zone E  タイムライングリッド                          │                 │
│          (時間ガター + スタッフ列 + 予約ブロック)       │                 │
│                                                      │                 │
└──────────────────────────────────────────────────────┴──────────────────┘
```

| 구역 | CSS 클래스 | 역할 | 크기 |
|------|-----------|------|------|
| **Zone A** | `.zone-a` | 앱 전역 헤더. 사이드바 토글·로고·스태프 정보·시각 | H 44px · 전체 폭 |
| **Zone B** | `.zone-b` | 캘린더 탐색. 날짜 이동·뷰 전환·예약 추가 | H 46px · 좌측 폭 |
| **Zone C** | `.zone-c` | 스태프 표시 필터. 드롭다운 셀렉터 | H 38px · 좌측 폭 |
| **Zone D** | `.zone-d` | 스태프 열 레이블 (일 뷰에서만 표시) | H 34px · 좌측 폭 |
| **Zone E** | `.zone-e` | 타임라인 그리드. 핵심 예약 조작 영역 | flex:1 · 스크롤 |
| **Zone F** | `.zone-f` | 보조 정보 패널. Zone A 아래부터 전체 높이 | W 210px |
| Zone F-1 | `.zone-f1` | 미니 캘린더 | ~ |
| Zone F-2 | `.zone-f2` | 일차 통계 | ~ |
| Zone F-3 | `.zone-f3` | 웨이팅 리스트 | flex:1 |

> **레이아웃 구조**: Zone A가 1행 전체 차지. 그 아래부터 `cal-wrapper`가 `[cal-left (Zone B~E)] | [Zone F]` 2열로 분리.  
> **Zone F는 Zone B와 같은 높이에서 시작** (툴바 아래가 아닌 탑바 바로 아래). 이로써 Zone F가 더 길어지고 웨이팅 리스트 공간이 확보됨.

## 3. Zone B — カレンダーナビバー

**디자인**: 흰색 배경 / Concept B 스타일 세그먼트 탭  
**구성 (좌→우)**:

| 요소 | 동작 |
|------|------|
| ‹ / › 버튼 | `navigate(-1)` / `navigate(1)`. 현재 뷰에 따라 1일/1주/1월 이동 |
| 今日 버튼 | `goToday()`. 2026-06-19로 이동 |
| 날짜 레이블 `#calDate` | 뷰별 형식: 日→`6月19日(木)` / 週→`6月16日〜22日` / 月→`2026年6月` |
| 뷰 탭 [日·週·月·一覧] | `switchView()`. 활성 탭: 흰색 pill + 그림자 |
| (우측) 📊 버튼 | Zone F 열기/닫기 토글 (`toggleSide()`) |
| ＋ 予約追加 버튼 | 퍼플(`#8E44AD`). `07_booking_form`으로 이동 |

## 4. Zone C — スタッフセレクター

**디자인**: 흰색 배경 / "スタッフ" 레이블 + 드롭다운  
**드롭다운 동작**:

- 버튼 표시: `全スタッフ (3名)` 또는 선택된 이름 나열 (`Yuki、Mika` 등)
- 클릭 → 메뉴 펼침. 화면 다른 곳 클릭 → 닫힘
- 체크마크(✓)로 현재 선택 상태 표시

**선택 항목**:

| 항목 | 동작 |
|------|------|
| 全スタッフ | 전체 선택 |
| Yuki / Mika / Kenji | 개별 토글. 복수 선택 가능 |
| Hana | "本日休" 표시. 선택 불가 (클릭 시 토스트) |

- 전체 해제 시 자동으로 全スタッフ 복귀
- 드롭다운 닫힌 뒤 선택된 스태프는 Zone C 우측에 컬러 태그로 표시

## 5. Zone D — スタッフコラムヘッダー

**표시 조건**: 일(日) 뷰에서만 보임
**구성**: 50px 시간 거터(공백) + 현재 지점에 체크인된 스태프 수만큼 균등 열 + 担当未定 열 1개(항상 마지막)

각 열에 표시:
- 스태프 색 도트 (7px 원)
- 스태프 이름 + (타지점 소속이면) `ヘルプ` 배지
- 당일 예약 건수 (`N件`, 회색)
- 체크인~체크아웃 시간대 (`09:10〜13:00` 또는 `13:05〜在店中`)

### 5-1. 멀티지점 체크인 모델 (중요)

기존 "휴무/출근" 단순 토글 대신, **PIN 로그인 = 체크인** 모델을 사용한다.

| 동작 | 결과 |
|------|------|
| 디자이너가 어느 지점 단말기에서 PIN 로그인 | 그 지점에 체크인(시작시각 기록) |
| 다른 지점 단말기에서 재로그인 | 이전 지점 자동 체크아웃 + 새 지점 체크인 (별도 "이동" 버튼 없음) |
| 체크아웃 완료(예: Kenji 13:00) | 해당 스태프 열은 캘린더에 계속 표시되지만, 배정 메뉴(5-2) 대상에서는 제외 |

`STAFF[]`에 `off` 불리언 대신 `homeBranch`, `checkIn`, `checkOut`(null=재실중) 필드를 사용. 오늘 한 번도 체크인하지 않은 스태프는 캘린더에 노출되지 않음(컬럼 자체가 없음).

### 5-2. 担当未定 열 + 배정 메뉴

지점만 보고 담당자를 지정하지 않은 예약(`s:null`)은 항상 마지막 "担当未定" 열(사선 무늬, 점선 카드)에 표시된다. 클릭 시 현재 재실 중(체크아웃 안 한) 스태프 목록이 담긴 팝오버가 열리고, 선택하면 즉시 해당 스태프 열로 이동한다 — 헬프 온 디자이너가 도착 후 미배정 예약을 받아가는 흐름.

## 6. Zone E — タイムライングリッド

### 6-1. 시간 거터 (좌측 50px)
- 9:00–19:00 정시 레이블
- 정각선: `#EBEBEB` / 30분선: `#F4F4F2` (더 연함)
- 배경: `#FAFAFA`
- 1시간 = 40px

### 6-2. 스태프 열
- `filtBK()` 기준으로 선택된 스태프 열만 표시 (`flex: 1`로 균등 분배)
- 휴무 스태프 열: 사선 무늬 배경
- 각 열 오른쪽 0.5px 구분선

### 6-3. 예약 블록 — Concept B 카드 디자인

```
position: absolute
top: toTop(sh, sm)  →  (h-9)*40 + m*(40/60)  px
height: dur*(40/60) - 2  px  (최소 22px)

┌──┬─────────────────────────────────┐
│  │  田中 美咲           [Yuki]     │  ← bk-top
│  │  カット＋カラー                  │  ← bk-service
│  │  09:00〜10:00        ¥14,300   │  ← bk-meta
│  │  [完了]                         │  ← status-badge
└──┴─────────────────────────────────┘
↑ 4px accent bar (스태프 색)  ↑ 흰색 body (#fff)
```

**높이별 정보 표시 규칙**:

| 블록 높이 | 표시 항목 |
|-----------|-----------|
| 22px~ | 고객명 |
| 34px~ | + 스태프 배지 |
| 34px~ | + 메뉴명 |
| 52px~ | + 시간 범위 + 금액 |
| 72px~ | + 상태 배지 |

**ブロック 예약 (status: block)**:
- 사선 무늬 배경, 고객명 회색, accent bar 회색

### 6-4. NOW 라인 — Concept B 스타일
- 빨간 가로선 (`#EF4444`, 2px) + 좌측 원형 마커
- 라인 위에 `10:24` 텍스트 레이블 (작은 빨간 박스)
- 첫 번째 스태프 열(인덱스 0)에만 배치

### 6-5. 배경 교대색
- 짝수 시간대 행: `#FAFAF8` (미묘한 교차)

## 7. Zone F — サイドパネル

**토글**: Zone B의 📊 버튼으로 `width:0` 슬라이드 닫힘  
**시작 위치**: Zone A 바로 아래 (Zone B~C와 같은 높이에서 출발)

### Zone F-1 — ミニカレンダー
- 2026년 6월 달력 (5×7 그리드)
- 오늘(19일): 퍼플(`#8E44AD`) 배경
- 예약 있는 날: 인디고 굵은 숫자 (`has-bk`)
- 날짜 클릭 → `jumpDate()` → 일 뷰 이동

### Zone F-2 — 日次統計
| 항목 | 값 |
|------|-----|
| 総予約 | 13件 (인디고) |
| 完了 | 5件 (에메랄드) |
| 残り | 7件 (퍼플) |
| ノーショー | 1件 (레드) |
| 売上見込み | ¥128,700 |

### Zone F-3 — ウェイティングリスト
- 대기 고객 순번·이름·메뉴·소요 + 「確定」버튼
- 「確定」클릭 → 토스트 알림 (실제 구현 시 07_booking_form 연동)
- `flex:1`로 남은 공간 모두 사용, 내부 스크롤

## 9. 뷰 전환 동작

### 일(日) 뷰
- Zone D(스태프 헤더) + Zone E(타임라인) 표시
- 선택된 스태프 수만큼 열 생성

### 주(週) 뷰
- Zone D 숨김
- 7일 열 헤더 + 시간 그리드
- 오늘(19일) 열: `#FAFBFF` 배경 + 실제 예약 블록 표시 (다른 날은 빈 그리드)
- 예약 블록: 3px 스태프 컬러 바 + 고객명 + 스태프명 (간소화)

### 월(月) 뷰
- Zone D 숨김
- 6×7 그리드 (42셀)
- 셀: 날짜 + 예약 바 3개 + `+N件` 더보기
- 오늘: `#EEF2FF` 배경 / 타월: `#F9F9F7` 배경

### 일람(一覧) 뷰
- Zone D 숨김
- 리스트 스크롤 (`#F0F2F7` 배경)
- 카드: 스태프 컬러 바(3px) | 시간 | 고객명+메뉴 | 금액+스태프 배지 | 상태 배지
- ブロック 예약 제외
