# 03 · 계산(결제수단 선택) 화면 정의서

> **파일**: `design/mockups/ja/03_pos_payment.html`  
> **목적**: 결제 플로우 2단계. 확정된 주문금액에 대해 포인트 사용, 결제수단 선택, 현금 거스름돈 계산, 팁·영수증 옵션을 설정하고 최종 결제를 실행함.  
> **진입 경로**: `02_pos_order` → 「会計する」 버튼  
> **다음 화면**: 「会計を実行」 클릭 → `04_pos_complete`  
> **이전 화면**: 「← 注文に戻る」 클릭 → `02_pos_order`  
> **레이아웃**: 좌(300px 주문요약) / 중(flex:1 결제수단) / 우(240px 합계+실행)

---

## 화면 구조 다이어그램

```
┌──────────────────────────────────────────────────────────────────┐
│  글로벌 탑바                                                      │
├──────────────────────────────────────────────────────────────────┤
│  스텝바: ✓주문 › ●계산 › 완료                                     │
├────────────┬──────────────────────────────┬──────────────────────┤
│ 주문 요약  │  결제수단 선택               │  합계                │
│ 고객 정보  │  (현금/카드/QR/분할/외상)    │  팁·영수증           │
│ 포인트 사용│  현금: 키패드+거스름돈       │  포인트 적립 예고    │
│            │  분할: 수단별 금액 행        │  실행·되돌아가기     │
└────────────┴──────────────────────────────┴──────────────────────┘
```

**공통 스타일**:
- 스텝 숫자 원: 18×18px, border-radius 50%

- pending: 테두리만 `#bbb`, 텍스트 `#bbb`
- 화살표 구분: `›`, `#D0D0CC`

#### A-01-4 · 스텝바 우측 정보

### C-02 · 현금 키패드 섹션 (id: `cashSection`)

`m-cash` 선택 시만 표시. 다른 수단 선택 시 `display:none`.

#### C-02-1 · 받은 금액 입력

| 항목 | 값 |
|------|----|
| 타입 | TextField (`#cashInp`) |
| 폰트 | 18px / 500, 오른쪽 정렬 |
| 단위 | `¥` 좌측 레이블 |
| 파라미터 | `cashReceived: number (JPY)` |

#### C-02-2 · 빠른 금액 버튼 (3개)

| # | 라벨 | 동작 |
|---|------|------|

**스타일**: 가로 1/3씩, Secondary 스타일, 12px

#### C-02-3 · 숫자 키패드 (12키)

| 항목 | 값 |
|------|----|
| 타입 | 3×4 CSS Grid (div `.cash-keypad`) |
| 버튼 크기 | 세로 40px (패딩 10px) |
| 배경 | `white`, 테두리 `.5px solid #E5E5E0`, 보더 레이디어스 8px |
| 폰트 | 15px / 500 |
| hover | `background: #F4F4F0` |
| active | `background: #E8E8E4` |

| 위치 | 키 | 동작 |
|------|-----|------|
| 1행 | 1, 2, 3 | 숫자 추가 |
| 2행 | 4, 5, 6 | 숫자 추가 |
| 3행 | 7, 8, 9 | 숫자 추가 |
| 4행 | 000, 0, ⌫ | 000 추가 / 0 추가 / 마지막 삭제 |

**입력 로직**: `kpStr`(문자열)에 append 후 `parseInt` → `cashRaw` 갱신 → `calcChange()` 호출  
**최대 7자리** 제한 (¥9,999,999 이하)

#### C-02-4 · 거스름돈 표시바

| 항목 | 값 |

## E. JS 상태 관리

| 변수 | 타입 | 초기값 | 설명 |
|------|------|--------|------|
| `subtotal` | number | 17600 | 주문 소계 (02에서 전달) |
| `ptsUsed` | number | 0 | 사용할 포인트 수 |
| `cashRaw` | number | 20000 | 받은 현금 |
| `kpStr` | string | '20000' | 키패드 입력 문자열 |
| `currentMethod` | string | 'cash' | 선택된 결제수단 |

| 함수 | 역할 |
|------|------|
| `updateTotal()` | 포인트·팁 반영 후 합계 재계산 및 전체 UI 갱신 |
| `calcChange()` | 현금 거스름돈 계산 및 색상 갱신 |
| `useAllPts()` | 전액 포인트 사용 |
| `selectMethod(m)` | 결제수단 선택 및 관련 섹션 표시/숨김 |
| `setCash(val)` | 빠른 금액 버튼 처리 |
| `kpPress(d)` | 키패드 숫자 입력 |
| `kpDel()` | 키패드 삭제 |
| `toggleReceipt()` | 영수증 토글 |
| `addSplitRow()` | 분할결제 행 추가 |
| `execPayment()` | 결제 실행 (→ 04) |
| `goBack()` | 주문 화면으로 복귀 (→ 02) |
