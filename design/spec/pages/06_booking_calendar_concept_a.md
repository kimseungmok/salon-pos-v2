## 3. 주요 구성 요소

### 3-1. 스태프 컬럼 헤더
- 각 열 상단에 스태프 이름 + 색 도트 + 당일 예약 건수(`N件`) 표시
- 휴무 스태프는 opacity 0.35 + 사선 무늬 배경으로 구분
- 필터 칩에서 스태프를 끄면 해당 열이 사라짐 (flex로 자동 분배)

### 3-2. 시간 거터 (좌측 50px)
- 9:00–19:00 정시 표시
- 정각선: `#EBEBEB` / 30분선: `#F4F4F2` (더 연하게)
- 1시간 = 40px

### 3-3. 예약 블록 — 정보 표시 규칙

블록 높이에 따라 표시 정보가 늘어남 (공간이 허락하는 만큼):

| 높이 | 표시 항목 |
|------|-----------|
| 22px 이상 | 고객명 |
| 30px 이상 | + 메뉴명 |
| 46px 이상 | + 시간 범위·소요 시간 |
| 60px 이상 | + 금액 |

- 배경색 = 예약 상태 색상 (아래 참고)
- 우측 상단에 반투명 도트: 시각적 악센트
- 블록 클릭 → 07_booking_form (상세/편집)

### 3-4. 현재 시각 라인
- 빨간 가로선 (`#EF4444`, 2px) + 좌측 원형 마커
- 스태프 열 0번(Yuki 열)에만 표시

## 4. 뷰 전환

### 일(日) 뷰 — 메인
- 위 구조 그대로. 스태프 컬럼 병렬 배치.

### 주(週) 뷰
- 가로축 = 월~일 7일. 세로축 = 시간.
- 당일(水) 칸은 `#FAFBFF` 배경으로 강조.
- 각 날짜 헤더: 요일 + 날짜 + 스태프 색 도트 4개.
- 예약 블록은 해당 날짜 칸에 좌측 컬러 바 + 고객명만 표시 (공간 제한).

### 월(月) 뷰
- 6×7 그리드 (42셀). 셀 높이 동일.
- 셀 내부: 날짜 숫자 + 예약 바 최대 3개 + `+N件` 더보기.
- 오늘 = `#EEF2FF` 배경, 날짜 텍스트 `#3B5BDB`.
- 타월(他月) 셀 = `#F9F9F7` 배경.

### 일람(一覧) 뷰
- 날짜 헤더 → 예약 카드 리스트 (시간 오름차순).
- 카드: 시간 | 스태프 컬러 바(3px) | 고객명 + 메뉴 | 금액 + 스태프 배지 | 상태 배지.
- 블록(ブロック) 예약은 목록에서 제외.

## 5. 상태 색상 시스템

| 상태 | 색상 | 설명 |
|------|------|------|
| `done` 完了 | `#888` 회색 | 시술 완료 |
| `checkin` チェックイン | `#3B5BDB` 인디고 | 내점 확인됨 |
| `confirmed` 確定 | `#10B981` 에메랄드 | 예약 확정 |
| `pending` 仮予約 | `#D4AC0D` 황금 | 가예약·미확정 |
| `noshow` ノーショー | `#8E44AD` 퍼플 | 무단 미방문 |
| `block` ブロック | `#555` 다크그레이 | 휴식·회의·차단 |

## 6. 스태프 필터 동작

- **全て 활성**: 전 스태프 표시. 전체 칩 + 개별 칩 모두 강조.
- **개별 클릭**: 全て 해제 → 해당 스태프만 토글. 열이 줄어들며 나머지 열이 flex로 확장.
- **전부 해제 시**: 자동으로 全て 복귀.

## 8. 컨셉 A의 강점 vs 약점

**강점**
- 스태프 공백 슬롯이 즉시 보임 → 신규 예약 배치 판단이 빠름
- "이 시간에 Mika가 비어 있구나"가 열 스캔 한 번으로 파악
- 3–4명 살롱에서 팀 효율 관리에 최적

**약점**
- 스태프 수 × 시간대 조합으로 화면이 조각남 → 한 시간대에 무슨 일이 일어나는지 가로 스캔이 필요
- 예약 블록이 좁아서 긴 메뉴명이 잘림 (열 4개 기준)
- 스태프 5명 이상이면 열 너비가 매우 좁아짐
