# 13 · 勤怠ログ — 화면 정의서

> **파일**: `design/mockups/v2/ja/13_staff_shift.html`
> **목적**: 전 스태프·전 지점의 체크인/체크아웃을 날짜별로 한눈에 조회.
> **진입**: 사이드바 → スタッフ → (또는 별도 메뉴)
> **컨셉**: **출퇴근 전용화면(B2)**. `12_staff_detail`의 개인별 勤怠ログ 탭(B1)과 비교 검토용.

---

## 1. 구역 구성

| 구역 | 역할 |
|------|------|
| Zone B | 날짜 네비게이션(‹›今日) + 지점 필터(全店舗/지점명) |
| 現在の在店状況 카드 | 전 스태프 현재 위치/체크인시각/헬프여부를 그리드로 한눈에 |
| 履歴 카드 | 시간순 체크인·체크아웃 이벤트 전체 목록 |
| Zone D | 총출근인원/현재재실/헬프건수/점포별 인원 |

## 2. 데이터 모델

`EVENTS[]`: `{time, name, branch, action:'in'|'out'}`. 각 스태프의 **최신 이벤트**로 현재 위치·재실여부를 역산(`computeStatus()`). 같은 시각에 두 스태프가 지점을 교차 이동하는 경우(예: Kenji↔Rio)도 시간순 정렬로 자연스럽게 표현됨.

## 3. B1 vs B2 비교 포인트

| | 01+06+22 (B1, 내재형) | 13 (B2, 전용화면) |
|---|---|---|
| 진입 비용 | 추가 화면 없음, 로그인/캘린더/개점 화면에 자연히 녹아있음 | 별도 메뉴 진입 필요 |
| 조회 범위 | 그 화면이 다루는 범위만(한 지점, 오늘) | 날짜·지점 자유 조회, 전 스태프 시계열 |
| 적합 대상 | 현장 스태프의 즉시 확인 | 매니저/오너의 사후 검토·정산 |
