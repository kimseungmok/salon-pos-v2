# 09 · 顧客一覧 — 화면 정의서

> **파일**: `design/mockups/v2/ja/09_customer_list.html`
> **목적**: 전체 고객 검색·필터·정렬 + 신규 고객 등록. `00_feature_overview.md` 4장(고객) 기준.
> **진입**: 사이드바 → 顧客管理
> **이동**: 고객 카드 클릭 → `10_customer_detail` / 予約 버튼 → `07_booking_form`
> **컨셉**: 06/08/05와 동일한 마스터+사이드 패턴 (좌측 리스트 · 우측 통계)

---

## 1. 구역 명명 (Zone Map)

```
┌─────────────────────────────────────────────────────────────────────────┐
│  Zone A  グローバルトップバー                                            │
├──────────────────────────────────────────────────────┬──────────────────┤
│  Zone B  ツールバー (탭 · 검색 · 정렬 · 추가)            │                 │
├──────────────────────────────────────────────────────┤  Zone D         │
│                                                      │  サイドパネル    │
│  Zone C  顧客リスト (카드 그리드/리스트)                 │  今日の状況     │
│                                                      │  태그 범례      │
└──────────────────────────────────────────────────────┴──────────────────┘
```

---

## 2. Zone B — ツールバー

| 요소 | 설명 |
|------|------|
| 탭 | `すべて` `VIP` `休眠` `ブラック` `今月誕生日` — 세그먼트 탭 |
| 검색 | 이름/전화번호 |
| 정렬 | 드롭다운: 来店頻度順 / 最終来店順 / 名前順 |
| ＋新規顧客登録 | 인라인 추가 폼 토글 (08의 패턴 재사용) |
| CSV出力 | 토스트만 (실제 구현 보류) |

---

## 3. Zone C — 顧客リスト

각 카드: 아바타(이니셜+색상) · 이름 · 전화번호 · 태그뱃지(VIP/休眠/ブラック) · 최근방문일 · 방문횟수 · 포인트 · 생일(이번달이면 🎂 강조)

### 태그 색상

| 태그 | 색상 | 기준 |
|------|------|------|
| VIP | `#D4AC0D` | 누적 방문 10회 이상 또는 수동 지정 |
| 休眠 | `#999` | 최근 방문 90일 이상 경과 |
| ブラック | `#E74C3C` | 노쇼 누적 또는 수동 지정 |
| 一般 | 표시 없음 | 기본 |

---

## 4. Zone D — サイドパネル

| 항목 | 설명 |
|------|------|
| 총 고객수 | 전체 카운트 |
| VIP / 休眠 / ブラック | 탭별 카운트 |
| 今月誕生日 | 이번달 생일 고객 수 |
| 평균 방문주기 | mock 고정값 |

---

## 5. JS 상태 관리

| 변수/함수 | 역할 |
|-----------|------|
| `CUSTOMERS[]` | 고객 데이터 (id, name, phone, tag, lastVisit, visitCount, points, birthdayMonth) |
| `curTab` / `filterText` / `sortBy` | 탭·검색·정렬 상태 |
| `switchTab` / `onSearch` / `onSort` | 갱신 함수 |
| `openDetail(id)` | 클릭 시 `10_customer_detail`로 이동 (mock: 토스트) |
| `toggleAddForm` / `submitAddForm` | 신규 고객 인라인 등록 (08 패턴 재사용) |
