# 23 · 閉店処理 — 화면 정의서

> **파일**: `design/mockups/v2/ja/23_store_close.html`
> **목적**: 영업 종료 후 매출 요약 확인 + 시재 마감 카운트(현금 검증) + 마감 체크리스트.
> **진입**: 사이드바 → 店舗設定 → 閉店処理
> **이동**: 閉店確定 클릭 → 영업일 종료 처리 (실제 구현 시 익일 `22_store_open`의 前日締め額에 인계)
> **컨셉**: 22_store_open과 동일한 2열 구조(체크리스트 / 시재 카운트) + 매출 요약 카드 추가

---

## 1. 구역 명명 (Zone Map)

```
┌─────────────────────────────────────────────────────────────────────────┐
│  Zone A  グローバルトップバー                                            │
├─────────────────────────────────────────────────────────────────────────┤
│  Zone B  ツールバー (날짜 · 閉店確定 버튼)                                │
├──────────────────────────────────┬──────────────────────────────────────┤
│  Zone C  売上サマリー＋チェック     │  Zone D  レジ締めカウント             │
│  Zone C-1 本日の売上サマリー        │  권종별 수량 입력 → 합계              │
│  Zone C-2 閉店チェック              │  예상 마감액과 차액 비교               │
└──────────────────────────────────┴──────────────────────────────────────┘
```

---

## 2. Zone C-1 — 本日の売上サマリー

| 항목 | 설명 |
|------|------|
| 総売上 | 05_transaction_history 기준 오늘 매출(환불 반영) |
| 取引件数 / 返金件数 | 〃 |
| 決済手段別 | 現金 / カード / QR決済 매출 분리 표시 (현금 매출만 시재 계산에 사용) |

---

## 3. Zone C-2 — 閉店チェックリスト

| # | 항목 |
|---|------|
| 1 | 본일 매출 정산서 출력·확인 |
| 2 | 시술 공간·대기 공간 정리 |
| 3 | 소모품·재고 마감 확인 |
| 4 | 출입문·창문 잠금, 보안 설정 ON |

---

## 4. Zone D — レジ締めカウント (핵심)

| 항목 | 설명 |
|------|------|
| 권종별 입력 | 22_store_open과 동일한 9종 권종 테이블 |
| 合計 | 입력 합산 |
| **予想締め額** | `開始金(¥50,000) + 本日現金売上 − 本日現金返金` 자동 계산 |
| **差額** | 합계−예상액. 0=일치(녹색), 그외 적색 경고 |

閉店確定 버튼은 체크리스트 4항목 전부 체크 + 시재 합계 입력(>0) 시 활성화. 차액이 있어도 진행 가능하되 경고 문구 유지(실제 운영에서는 사유 기록 필요 — 추후 확장 포인트로 명시).

## 5. JS 상태 관리

| 변수/함수 | 역할 |
|-----------|------|
| `TODAY_SALES` | 매출 요약 mock 데이터 (총매출, 거래건수, 환불건수, 결제수단별 금액) |
| `CHECKLIST[]` | 마감 체크리스트 |
| `DENOMS[]` | 권종별 입력 (22와 동일 구조) |
| `START_CASH` | 금일 개점 시 확정된 시작금 (mock 고정값, 실제 구현 시 22에서 인계) |
| `expectedClose()` | 시작금 + 현금매출 − 현금환불 계산 |
| `toggleCheck(id)` / `updateQty(value, qty)` | 22와 동일 |
| `canClose()` / `confirmClose()` | 마감 확정 조건 판정 및 처리 |
