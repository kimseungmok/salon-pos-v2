# 10 · 顧客詳細 — 화면 정의서

> **파일**: `design/mockups/v2/ja/10_customer_detail.html`
> **목적**: 고객 1인의 전체 정보·시술 기록(카르테)·이력 조회 및 액션. `00_feature_overview.md` 4장 기준.
> **진입**: `09_customer_list` 카드 클릭
> **이동**: 予約 → `07_booking_form` / 会計 → `02_pos_order`
> **컨셉**: 토스포스에 없는 살롱 특화 차별점 — **カルテ(시술 기록·컬러 레시피)** 탭이 핵심.

---

## 1. 구역 명명 (Zone Map)

```
┌─────────────────────────────────────────────────────────────────────────┐
│  Zone A  グローバルトップバー                                            │
├─────────────────────────────────────────────────────────────────────────┤
│  Zone B  ヘッダー (戻る · 顧客名 · タグ · アクション)                     │
├─────────────────────────────────────────────────────────────────────────┤
│  Zone C  プロフィールサマリー (가로 전체)                                │
├─────────────────────────────────────────────────────────────────────────┤
│  Zone D  タブ (基本情報 / カルテ / 履歴)                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

---

## 2. Zone B — ヘッダー

| 요소 | 설명 |
|------|------|
| ‹ 顧客一覧 | 09로 복귀 |

| VIP指定 / ブラック指定 | 토글, 즉시 Zone C 배지 갱신 |

## 3. Zone C — プロフィールサマリー

| 항목 | 설명 |
|------|------|
| 아바타·이름·전화번호 | |
| 생일 | 이번달이면 강조 |

---

## 4. Zone D — タブ

### 4-1. 基本情報
- 연락처, 생일, 주소(선택), 소개인, 메모 — 編集 버튼으로 인라인 수정 토글

### 4-2. カルテ (핵심 차별점)
- **시술 기록 카드** (날짜순): 시술명, 담당 스태프, 사용 약제/컬러 레시피, 시술 메모, 사진 placeholder
- **컬러 레시피 보관**: 이전 사용 배합(브랜드/톤/비율) 카드 형태로 별도 보관 → 다음 시술 시 "동일 레시피 복제" 가능
- 태그·메모 추가 (두피 상태, 알러지 등 주의사항)

### 4-3. 履歴
- 통합 타임라인: 시술/구매/예약/포인트 적립·사용/외상 항목을 시간순으로 한 리스트에 표시, 유형별 아이콘·색상 구분

## 5. JS 상태 관리

| 변수/함수 | 역할 |
|-----------|------|
| `CUSTOMER` | 상세 대상 고객 객체 (mock: 09의 佐藤里奈 고정) |
| `KARTE[]` | 시술 기록 배열 (date, menu, staff, recipe, memo) |
| `COLOR_RECIPES[]` | 보관된 컬러 레시피 |
| `HISTORY[]` | 통합 이력 (type, date, label, amount) |
| `curTab` | `'basic'\|'karte'\|'history'` |
| `switchTab(t)` | 탭 전환 |
| `toggleVip()` / `toggleBlack()` | 태그 토글 |
| `duplicateRecipe(id)` | 레시피 복제 → 새 시술기록 작성 폼에 자동 입력 (mock: 토스트) |
