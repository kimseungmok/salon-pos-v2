# 12 · スタッフ詳細 — 화면 정의서

> **파일**: `design/mockups/v2/ja/12_staff_detail.html`
> **목적**: 스태프 1인의 기본정보(읽기 전용)·근태(체크인-아웃 로그)·매출을 한 화면에서 조회.
> **진입**: `11_staff_list` 카드 클릭 (`?id=` 쿼리로 대상 식별)
> **컨셉**: POS 화면. 계정 발급/PIN/권한레벨 변경은 **勤怠管理サイト**(통근관리사이트, 관리자 전용)로 이관됨 — 2026-06-21 결정.

---

## 1. 구역 구성

| 구역 | 역할 |
|------|------|
| Zone B | 戻る · 이름+직급배지 · PIN재발행/無効化/保存 액션 |
| Zone C | 아바타·연락처·소속·입사일 + 이번달매출/건수 |
| Zone D | 탭: 基本情報・権限 / 勤怠ログ / 売上 |

## 2. 탭별 내용

### 基本情報・権限
ログインID(읽기전용), 연락처, 소속점, 입사일, 권한레벨(읽기전용 배지). **PIN 표시, 권한레벨 변경, 권한 매트릭스는 모두 제거됨** — 하단에 "ログインID・PIN・権限レベルの変更は勤怠管理サイト（管理者専用）で行います" 안내문만 표시. PIN은 6자리, 발급/재발급도 통근사이트 책임.

### 勤怠ログ
멀티지점 체크인/체크아웃 이력. `06_booking_calendar`/`22_store_open`과 동일한 체크인 모델 데이터를 재사용. 타지점 기록은 `ヘルプ` 배지.

### 売上
최근 6개월 매출 막대그래프.

## 3. 계정/권한 관리 범위 이관 (2026-06-21)

기존엔 12에 권한레벨 변경 select + 편집 가능한 권한 매트릭스 + PIN 표시 토글이 있었으나, **계정 제어 관리는 통근관리사이트의 역할**로 결정되어 POS(12·33 모두)에서 제거함. `33_staff_accounts.html`은 이 결정 이전에 만든 A2(전용화면) 비교안으로, 현재는 **참고용/사용 안 함** — 자세한 내용은 `33_staff_accounts.md` 참조.
