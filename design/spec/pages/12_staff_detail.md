# 12 · スタッフ詳細 — 화면 정의서

> **파일**: `design/mockups/v2/ja/12_staff_detail.html`
> **목적**: 스태프 1인의 계정정보(ID/PIN/권한)·근태(체크인-아웃 로그)·매출을 한 화면에서 관리.
> **진입**: `11_staff_list` 카드 클릭 (`?id=` 쿼리로 대상 식별)
> **컨셉**: **계정관리 통합형(A1)**. 별도 화면(`33_staff_accounts`, A2)과 비교 검토용.

---

## 1. 구역 구성

| 구역 | 역할 |
|------|------|
| Zone B | 戻る · 이름+직급배지 · PIN재발행/無効化/保存 액션 |
| Zone C | 아바타·연락처·소속·입사일 + 이번달매출/건수 |
| Zone D | 탭: 基本情報・権限 / 勤怠ログ / 売上 |

## 2. 탭별 내용

### 基本情報・権限
ログインID, PIN(마스킹+표시 토글), 연락처, 소속점, 권한레벨 select. 하단에 **읽기 전용** 권한 매트릭스(4단계×6기능), 현재 직급 컬럼 강조 표시.

### 勤怠ログ
멀티지점 체크인/체크아웃 이력. `06_booking_calendar`/`22_store_open`과 동일한 체크인 모델 데이터를 재사용. 타지점 기록은 `ヘルプ` 배지.

### 売上
최근 6개월 매출 막대그래프.

## 3. A1 vs A2 비교 포인트

| | 11/12 (A1, 통합형) | 33 (A2, 전용화면) |
|---|---|---|
| 진입 동선 | 스태프 한 명씩 클릭해서 확인 | 전체 계정을 테이블 한 화면에서 일괄 관리 |
| 권한 매트릭스 | 읽기 전용(해당 스태프 권한만 강조) | 토글로 직접 ON/OFF 편집 가능 |
| 근태/매출 | 같은 화면에 탭으로 포함 | 없음 (13_staff_shift가 별도 담당) |

## 4. 관련 파일

| 파일 | 역할 |
|------|------|
| `12_staff_detail.html` | 이 화면 |
| `11_staff_list.html` | 진입 출처 |
| `13_staff_shift.html` | 근태 전용 화면 (B2, 비교용) |
| `33_staff_accounts.html` | 계정+권한 전용 화면 (A2, 비교용) |
