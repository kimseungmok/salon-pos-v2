# 17 · 売上ダッシュボード — 화면 정의서

> **파일**: `design/mockups/v2/ja/17_reports_dashboard.html`
> **목적**: 매출 현황을 기간·지점별로 한눈에 조회. KPI·추이·결제수단·스태프/메뉴 랭킹·지점 비교를 한 화면에 모음.
> **진입**: `01_login_main` 런치패드 「売上状況」 타일
> **이동**: 로고 클릭 → `01_login_main`으로 복귀

---

## 1. 구역 구성

| 구역 | 역할 |
|------|------|
| Zone B | 기간 탭(今日/今週/今月) + 지점 select(全店舗/渋谷店/池袋店) + 기간 라벨 |
| KPI 4종 | 총매출(반환반영후)·거래건수·객단가·반환액, 전기간 대비 ▲▼ |
| 売上推移 | 기간별 추이 바차트(오늘 데이터=남색 강조, 그 외=골드) |
| 決済手段別 比率 | 현금/카드/PayPay/LINE Pay/후불 비중 바 (05/23과 동일 색상 체계 재사용) |
| スタッフ別売上ランキング | 매출 내림차순, 1~3위 순위 색상 강조 |
| 施術メニュー別売上 TOP5 | 메뉴별 매출 내림차순 바 랭킹 |
| 店舗別比較 | 渋谷店 vs 池袋店 매출 비교, 본기간 1위에 `lead` 하이라이트 |

## 2. 데이터 모델

`DATASETS = {today, week, month}` — 각 기간별로 `sales/count/refund/prevSales/trend[]/byMethod{}/byStaff[]/byMenu[]/byBranch{}`를 독립적으로 보유한 mock. 지점 필터는 새 데이터셋을 만들지 않고 `branchFactor()`로 `byBranch` 비중만큼 전체 숫자를 스케일링하는 단순화된 방식(실제 구현 시엔 지점별 원본 집계로 대체 필요).

## 3. 멀티지점 연계

이 화면의 店舗別比較 카드는 이번 세션에서 도입한 멀티지점 운영 모델(06/12/13과 동일 인물·지점 체계: 渋谷店/池袋店, Yuki/Mika/Kenji/Rio)을 그대로 재사용. 스태프별 랭킹도 동일 인물 데이터를 사용해 다른 화면과 일관성 유지.
