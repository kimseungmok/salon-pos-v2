# 01 · 로그인 / 메인(런치패드) 정의서

> **파일**: `design/mockups/ja/01_login_main.html`  
> **목적**: 앱의 첫 진입점. 스태프가 자신을 선택하고 PIN으로 인증한 후 메인 런치패드로 이동. 자동 잠금 후 복귀 화면이기도 함.  
> **진입 경로**: 앱 최초 실행, 🔒 잠금 버튼 클릭, 자동 잠금 타임아웃  
> **다음 화면**: 인증 성공 → 메인 런치패드 (같은 파일 내 화면 전환)

---

## 화면 구성 개요

```
[로그인 화면]  PIN 인증 성공  [메인 런치패드]
     ↓  ──────────────────→       ↓
스태프 선택                   🔒 클릭 시 복귀
PIN 입력
```

두 화면이 하나의 HTML 파일 안에서 `display` 전환으로 구현됨.

## C. 화면 전환 로직

| 이벤트 | 동작 |
|--------|------|
| 스태프 선택 | `selectedStaff` 변수 갱신, 선택 표시, PIN 입력 활성화 |
| PIN 4자리 입력 | 자동으로 확인 (→ 버튼 불필요) |
| PIN 일치 | `loginScreen.style.display = 'none'`, `mainScreen.style.display = 'flex'` |
| PIN 불일치 | shake 애니메이션, dot 초기화 |
| 🔒 클릭 | 반대로 전환, `selectedStaff` 초기화, PIN dot 초기화 |
| 자동 잠금 | 5분 무조작 시 자동 로그인 화면 복귀 (구현 예정) |
