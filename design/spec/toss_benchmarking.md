## 0. 전체 결론

토스플레이스 실제 화면은 우리가 처음 추측했던 것보다 **훨씬 미니멀**하다. 화려한 차트나 다단 탭 대시보드는 거의 없고, "숫자 몇 개 + 리스트 + 큰 버튼 1~2개" 조합이 대부분. 반면 **메뉴/상품 타일은 화이트카드가 아니라 단색 파스�일 배경 전체채움**이고, **햄버거 메뉴는 화면폭의 절반 이상을 차지하는 2열 그리드**라는 점은 예상과 달랐다.

| 항목 | 처음 추측 | 실제 확인된 것 |
|------|----------|---------------|
| 상단바 색 | 토스블루(#0064FF) 풀톤일 것 | **다크 네이비/슬레이트**, 블루는 버튼·강조에만 |
| 메뉴/상품 카드 | 화이트카드+컬러스트립 | **단색 파스텔 배경 전체채움**, 흰 텍스트 |
| 햄버거 메뉴 폭 | 작은 사이드바(~22%) | **화면폭의 ~55%**, 2열 아이콘 그리드 |
| 매출 리포트 | 차트·랭킹 다수의 대시보드 | **숫자 3개(실매출/주문건수/반품)+결제수단 리스트**뿐, 그래프 없음 |
| 직원 관리 | 권한레벨·PIN 관리 화면 있을 것 | **없음** — "매출장부 연동할 직원 초대"(이름+연락처)뿐 |
| 상품 등록 | 사진 업로드 | **색상 스와치 선택**(사진은 별도 선택 항목) |

## 1. 화면별 상세 대조 기록

### 1-1. 주문/결제 화면 (02/03/04)
**출처**: `guide/sector/service-store/payment/payment`

- 메뉴 그리드: 카테고리 탭(흰 배경, 밑줄로 활성표시) 아래 상품 타일. **타일은 카테고리 등록 시 고른 색상 스와치가 배경 전체**, 이름(좌상단)·가격(좌하단)·시간 표시는 모두 흰 텍스트. 품절 상품은 우하단에 검정반투명 "품절" 배지.
- 우측 카트 패널: 상단에 "매장/포장/둘다" 세그먼트(살롱 무관 — 외식업 전용 기능, 미적용), 아이템 리스트(수량 ±, 메뉴명, 가격), 하단 "할인"·"분할결제" 버튼 + 전체폭 블루 "○○원 결제" 버튼.
- 결제수단 선택 화면은 가이드에 명시적 스크린샷 없음 — 카드는 단말기에 꽂으면 자동 결제, 현금은 "결제수단으로 현금 선택" 정도로만 텍스트 설명. 별도의 화려한 방법-선택 그리드 화면은 확인되지 않음.
- 적용: `02_pos_order.html`에 TILE_COLORS 팔레트로 단색타일 적용 완료. `03/04`는 구조적 근거가 약해 기존 카드형 유지(추가 단서 발견 시 갱신).

### 1-2. 예약 캘린더 (06)
**출처**: `guide/sector/service-store/booking`

- 좌측: 월간 미니캘린더(일요일 빨강, 토요일 파랑, 선택일 진한 파랑 원).
- 우측: 담당자별 컬럼(컬러닷+이름), 예약 블록은 **스태프별 단색 파스텔 배경 전체채움**(화이트바디+컬러스트립 아님), 블록 안에 이름(볼드)+시술명, 노쇼는 "노쇼" 작은 칩.
- 현재시각 라인: 파란 가로선 + 시각 라벨.
- 클릭 시 팝업: 아바타+이름+"예비단골,총N번 방문,노쇼N회"+포인트/시술정보+아이콘 메타(담당자/일시/메모)+우하단 floating "+" 버튼.
- 적용: `06_booking_calendar.html`에 `toPastel()` 함수로 스태프 컬러를 백색 62% 혼합한 파스텔로 변환, 블록 전체배경 적용 완료.

### 1-3. 고객관리 (09/10)
**출처**: `guide/sector/service-store/crm/customer-management`

- 고객 그룹은 **4가지, 직접 설정 불가(자동 판정)**:
  - 첫방문: 처음 방문
  - 예비단골: 2회 이상 방문, 아직 단골 아님
  - 단골: **3개월간 7회 이상 방문**(업종별 기준 다름 — 카페는 한달 평균 2회 이상)
  - 뜸한고객: 예비단골/단골이면서 **45일간 미방문**
- 리스트 행: "그룹|이름" + 마스킹 전화번호, 총방문/한달간방문, 선불권 금액, 포인트.
- 상세화면: 戻る + 아바타 + "이름(마스킹전화)" + 그룹·방문 요약 한 줄 + 수정버튼. **항상 보이는 메모 입력란**("사장님만 볼 수 있어요"). 포인트(−/+)/선불권(+) 2칸 카드. 탭: 분석/적립내역/선불권내역/쿠폰내역.
- 적용: `09_customer_list.html`에 `groupOf()` 동적 판정 함수로 전면 교체, `10_customer_detail.html` 헤더를 토스 패턴으로 교체(카르테/시술기록 등 살롱 고유 탭은 보너스로 유지).

### 1-4. 직원 관리 (11/12/13/33)
**출처**: 가이드 전체 검색 — "직원", "권한관리" 키워드로 검색해도 권한/PIN 관리 화면 자체가 존재하지 않음. 유일한 관련 기능: `guide/sector/prepaid-store/payment/connect-sajangnim-book` "매출장부 연동할 직원 초대하기".

- 직원 초대 = 이름+휴대폰번호 입력 → 알림톡 발송 → 동의하면 매출(사장님 매출장부 앱) 연동.
- 상태는 **대기중 / 연결됨** 두 가지뿐. 권한레벨, PIN, 역할 구분 없음.
- 적용: `33_staff_accounts.html`을 PIN표시·권한매트릭스 전부 제거하고 토스의 초대 리스트+모달로 단순화. `12_staff_detail.html`은 이미 "PIN·권한 변경은 통근관리사이트 전용"으로 정리돼 있어 추가수정 불필요. `11/13`은 토스에 대응 화면이 없어 우리 자산 유지.

### 1-5. 상품/재고 (25/26, 14/15)
**출처**: `guide/start-here/product-category-setup`

- 상품관리 좌측 서브탭: 상품/옵션/카테고리/할인(라이트그레이 1열 리스트, 선택시 파란 텍스트+좌측 파란바).
- "상품 추가" = **모달**(전체페이지 아님): 색상 스와치 11개(백색 포함) + 미리보기 박스, 키오스크노출 토글, 상품이름, **카테고리 선택은 칩 단일선택**(+새카테고리등록 링크), 가격+"매번 직접 입력할게요" 체크(시가설정), **옵션은 칩 다중선택**.
- 카테고리 관리: 리스트 + 토글(키오스크노출) + 상단 "순서 편집"/"영어표기"/"+카테고리 추가"(블루 솔리드).
- 적용: `25_product_list.html`에 모달 신설(색상스와치+칩), `26_product_form.html`은 25로 리다이렉트하는 호환파일로 축소.

### 1-6. 매출 리포트 (17)
**출처**: `guide/product/terminal/check-sales-report`

- 상단: 일간/주간/월간 세그먼트(다크필 선택) + 날짜 네비게이션(‹조회일›).
- "오늘 오후 OO시 기준 총 데이터예요" 안내문.
- 핵심 숫자 3개만: **실매출**(크게), **주문건수**, **반품**(빨강, 음수).
- "결제수단별 매출" 리스트(현금/카드/QR코드·바코드, 아이콘+라벨+금액).
- 하단 버튼: "결제 내역 보기"(아웃라인) + "출력하기"(블루 솔리드).
- 차트·그래프·랭킹·전년비교 **전혀 없음**. 매출달력은 별도 화면(달력 그리드에 일별 매출 숫자만, 날씨·메모 기능은 이번 조사에서는 확인되지 않음 — 이전 회차에 잘못 추정했던 부분, 정정 필요).
- 적용: `17_reports_dashboard.html`의 売上概況 탭 최상단에 토스 그대로의 단순 레이아웃을 배치하고, 기존에 만든 추이차트·전년비교·메뉴랭킹·스태프실적·캘린더·점포비교는 "토스 기본화면에 없는 보너스 정보"로 명확히 라벨링하여 하단에 유지.

> **정정사항**: 이전 세션 노트에서 "매출달력에 날씨·메모 기능"이라고 적었던 것은 §1-3(고객관리, 포인트·스탬프 가이드)의 다른 화면과 혼동한 기록으로 보임. 매출달력 자체는 날짜 그리드 + 일별 매출 숫자만 확인됨. 추후 재확인 필요.

### 1-7. 시재 관리 (22/23)
**출처**: `guide/sector/service-store/payment/cash-management`

- 좌측 서브탭: 시재관리(현재 돈통금액)/현금정산내역/설정.
- "현재 돈통금액" 화면: **총액 큰 숫자 표시** + 우측 공유 숫자패드(포커스된 입력칸에 입력) + 좌측 권종별 입력 그리드 — **지폐(10만/5만/1만원)는 "장" 단위, 동전(500/100/50/10원)은 "개" 단위**로 구분 표시. +/- 버튼은 없음(텍스트 입력만).
- 기능: "돈 넣기"/"돈 빼기"/"돈통 정산" 3가지.
- 적용: `22/23`에 "総額" 큰 숫자 표시 카드 추가, 권종별 단위(枚/個) 라벨 추가. 사용자가 명시적으로 요청했던 +/-버튼(혼자 개점하는 직원 편의)은 토스엔 없지만 보너스 기능으로 유지.

### 1-8. 쿠폰 발행 (19)
**출처**: `guide/sector/service-store/crm/coupons`

- "시즌널(이벤트) 쿠폰" = 매장 방문했던 고객에게 발송하는 1회성 쿠폰. **마케팅/광고 수신 동의한 고객에게만** 발송 가능.
- 만들기 STEP1: 시즌 템플릿 아이콘 선택(화이트데이/벚꽃/졸업/눈오는날/발렌타인/생일/크리스마스/설날/어린이날/어버이날/시험기간/할로윈/비오는날/무더위 등) — **현재 제공되는 기본 템플릿 내에서만** 가능.
- STEP2: 혜택 = **할인**(금액 또는 %) 또는 **증정**(상품 지정, 여러개 선택시 가장 비싼 상품 1개만 무료) 중 택1. 할인 적용범위(총주문금액/특정상품) + 최소주문금액.
- STEP3: 유효기간(7일/14일/30일/상시) 토글.
- 적용: `19_coupon_management.html`의 "+クーポン発行" 버튼을 토스 그대로의 3-step 모달로 교체(시즌 아이콘그리드+혜택토글+유효기간토글). 기존 영수증쿠폰번호(로또방식/할인비밀번호)는 토스§3-4/4-5 별개 기능이라 그대로 유지.

### 1-9. 포인트 적립 (21)
**출처**: `guide/sector/service-store/crm/points`

- 포인트 적립 ON/OFF 토글 + 4개 설정값만 존재: **①적립률(%)** ②최소사용포인트(P) ③적립가능상품(전체상품/일부제외) ④포인트 사용가능상품(전체상품/일부제외). 그 이상의 세부 규칙(등급/티어, 포인트 가치환산, 유효기간)은 토스 기본화면에 없음.
- 적용: `21_point_policy.html`을 토스 4항목 그대로의 카드로 재구성, 1포인트가치·유효기간·로열티티어는 "프레샤 참고"로 명확히 분리해 하단에 유지.

### 1-10. 캠페인 (20)
**출처**: `guide/sector/service-store/payment/discount`

- 확인 결과 토스에는 상시 "캠페인 관리" 화면이 **없음**. 마케팅 관련 기능은 ① 결제 시 즉석 "할인"(메뉴 선택 후 적용, "자주 쓰는 새 할인" 등록 가능, "자동 할인" 설정도 가능하지만 카트 화면의 즉석 액션일 뿐 상시 대시보드 아님) ② 시즌쿠폰(19) ③ 포인트(21) 세 가지로 완결.

### 1-11. 결제수단 선택·분할결제 (03)
**출처**: `guide/sector/service-store/payment/split-payment`

- 결제수단 선택 화면: "총 결제금액 표시" + "쿠폰·포인트·선불권 사용" 링크 + **결제방법 그리드(신용카드/현금/QR결제/계좌이체/전화예약 등 아이콘+라벨 카드)**.
- 분할결제 = 3-tab 세그먼트(**금액으로 결제 / 더치페이 / 메뉴별 결제**) → 결제수단 그리드 → 결제 완료 시 "OOOO원 결제완료" + "남은금액 OOOO원(총 OOOO원)" + "전자영수증 전송(N회가능)".
- 대조 결과: `03_pos_payment.html`이 **이미 거의 동일한 구조**(아이콘+라벨 결제수단그리드, 3탭 분할결제, 현금 받은금액·거스름돈 인라인계산)로 구현되어 있어 추가 수정 불필요. 결제수단 종류(後払い/掛け売り/回数券/利用券 등)는 토스보다 많지만 살롱 업종 특성상 합리적 확장이라 유지.
